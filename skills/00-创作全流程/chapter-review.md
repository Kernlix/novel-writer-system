# 🔎 章节审查 Skill

## 命令
`/novel:review`

## 功能
多维度审查已完成章节，生成结构化审查报告。

## 六道质量门禁

### 门禁 1：基础规范
- [ ] 无错别字、语病
- [ ] 标点符号使用正确（全角中文标点）
- [ ] 章节字数在2000-4000纯中文字符范围内
- [ ] 段落长度适中（不过长也不过碎）
- [ ] 文件名格式正确

### 门禁 2：叙事结构
- [ ] 有明确的开端/发展/结尾
- [ ] 开篇200字内有钩子
- [ ] 结尾有留白/悬念，非总结式收尾
- [ ] 每章至少有一个情感起伏
- [ ] 视角清晰一致，无混乱切换
- [ ] 时间顺序明确

### 门禁 3：角色逻辑
- [ ] 角色行为符合人设（无OOC）
- [ ] 角色对话有区分度（不看标签也能识别谁在说话）
- [ ] 角色情感变化有充分铺垫
- [ ] 角色行事动机合理

### 门禁 4：情节质量
- [ ] 本章对主线/支线有推进作用
- [ ] 有足够的冲突或张力
- [ ] 信息量适中（不灌水也不赶进度）
- [ ] 结尾有继续阅读的动力（悬念/期待/疑问）

### 门禁 5：语言质感（AI味检测）
- [ ] 无AI味高频词泛滥（然而/但是/因此/不过/突然/似乎）
- [ ] 无对称句式泛滥（既...又.../不仅...更...）
- [ ] 描写具体生动，非通用描写
- [ ] 情感通过动作或环境展示，非直接告知
- [ ] 对话与动作/神态自然穿插，避免纯对话墙
- [ ] 设定信息通过行动/对话展示，而非直接说明
- [ ] 叙事比例合理（叙述:独白:对话:描写 ≈ 4:3:2:1）

### 门禁 6：一致性（交叉检查）
- [ ] 与已有设定无矛盾
- [ ] 与前文衔接自然（无跳跃感）
- [ ] 与角色当前状态一致
- [ ] 伏笔使用/回收符合原计划
- [ ] 无时间线矛盾

## 输出格式

审查报告以 JSON 格式输出到 `.story-system/reviews/chapter_XXX.review.json`：

```json
{
  "chapter": "第XX章",
  "title": "章节标题",
  "rating": "X/10",
  "gates": {
    "basic": "✅ 通过",
    "narrative": "✅ 通过",
    "character": "✅ 通过",
    "plot": "✅ 通过",
    "language": "✅ 通过",
    "consistency": "✅ 通过"
  },
  "issues": [
    {
      "severity": "medium/minor",
      "item": "问题描述",
      "fix": "修改建议"
    }
  ],
  "summary": "一句话结论"
}
```

问题优先级：
- **🔴 高**：必须修复（设定矛盾/逻辑漏洞/严重影响阅读）
- **🟡 中**：建议修复（角色OOC/节奏问题）
- **🟢 低**：可改可不改（措辞优化/细节增强）
