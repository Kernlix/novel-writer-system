# 规则阶梯揭示 Skill

> **核心来源**：《十日终焉》第251-260章——地羊「四情扇」游戏的三层谎言揭示
> **提炼方法**：从死亡游戏机制、信息差与真相维度中提取可复用叙事技法
> **适用场景**：规则怪谈、死亡游戏、悬疑推理、设有复杂规则的博弈场景

---

## 核心原理

规则本身的真假是死亡游戏中最深刻的悬念。**当规则可能是谎言时，理解规则本身就是一场博弈。**

「规则阶梯揭示」技法指的是：游戏规则没有一次性完全揭露，而是分层次、由浅入深地揭示规则的真相——每一层揭示都改写了读者对整个游戏的理解。

```
第一层：表面规则 → 读者以为理解了游戏
   ↓
第二层：规则有隐藏条件 → 读者发现不对
   ↓
第三层：规则本身是谎言 → 读者认知被颠覆
   ↓
第四层：规则制定者也是玩家 → 整个游戏的前提被重构
```

---

## 技法1：规则的表面层——建立信任

游戏开始时公布的规则是**可信的但不完整**——不是谎言，是"说一半的真话"。

### 1.1 表面规则的设计原则

1. **规则本身必须合理**：第一眼看起来完整、公平、可执行
2. **规则必须有明确目标**：参与者知道"怎么做才能赢"
3. **规则必须有明确约束**：什么可以做、什么不可以做
4. **规则必须有奖励机制**：赢的人能得到什么

### 1.2 《十日终焉》案例

地羊公布的「四情扇」规则：
- 50名参与者，每人3把扇子（喜/怒/哀/乐随机）
- 两人配对凑齐四情可逃脱
- 每人3次配对机会
- 每半小时补充一次扇子
- 逃脱可获得10颗「道」

→ 读者（和参与者）都认为理解了游戏——配对合作就能赢。

### 1.3 表面规则的写作模板

```markdown
主持人公布规则：
"各位，本场游戏名为「XXX」。规则如下：
第一，每人获得三把扇子。
第二，两人配对凑齐四种花色可逃脱。
第三，每半小时补充一次扇子。
……祝各位好运。"

→ 参与者点头，读者放心——规则很清晰。
→ 但这个规则的每一个字都可能是"说了一半的真话"。
```

## 技法2：规则的隐藏条件层——制造危机

在规则执行过程中揭示**没有被说出来的条件**——规则本身没有变，但"执行规则"有了代价。

### 2.1 隐藏条件的设计方法

| 类型 | 说明 | 《十日终焉》案例 |
|:-----|:-----|:----------------|
| **隐藏成本** | 使用规则有隐性成本 | "补充扇子"需要3颗「道」购买 |
| **隐藏门槛** | 规则执行有隐性门槛 | 没有「道」就买不起扇子 |
| **隐藏时间** | 规则有时间维度 | 每30分钟扇子重置但需要付费 |
| **隐藏消耗** | 规则有消耗品 | 配对机会是不可再生的 |

### 2.2 隐藏条件的揭示时机

规则公布后不久——让角色在"以为自己理解规则"之后，突然发现"规则不是我想的那样"。

《十日终焉》案例：
- 第251章末尾：30分钟到，地羊竖起小黑板——"折扇每把售价三颗道"
- 齐夏终于明白了："每个人都有补充扇子的「机会」，可不见得大家都有补充扇子的「能力」"
- 这句话是对社会阶层的精辟总结——也是游戏第一层谎言的揭示

### 2.3 隐藏条件的写作模板

```markdown
半小时到了。
主持人拿出新的扇子。
参与者们走上前——但他们愣住了。
小黑板上写着：每把扇子售价 XXX 颗「道」。
公告上没有写"需要付费"——但也没有写"免费"。
规则没有说谎。规则只是没有说全。
```

## 技法3：规则数字的深层谎言层——颠覆认知

当参与者以为"规则有隐藏条件但整体可信"时，主角发现**规则本身的数据就是谎言**——连前提条件都是假的。

### 3.1 数字谎言的设计方法

这种反转需要一个数学上的"不可能"来触发：
- 扇子总数÷花色数=非整数 → 数据本身有矛盾
- 人数计算 → 实际人数与公布人数不符
- 时间计算 → 时钟速度和现实不符

### 3.2 《十日终焉》案例——51人的真相

齐夏的推理链条：
1. 四种花色一样多 → 150把扇子 ÷ 4 = 37.5把/花色 → **不可能**
2. 地羊不可能在花色数量上说谎（否则游戏无意义）→ 说谎的只能是参与者人数
3. 50人 → 实际是 **51人** → 多了一个隐藏的参与者
4. → 还有更多层的谎言：不止51人，地羊本人也是参与者

这个反转的冲击力来源于：**读者从开篇就相信的"50人"设定被彻底推翻。**

### 3.3 数字谎言的写作原则

1. **数字必须是公开的**：参与者和读者都知道这些数字
2. **矛盾必须是隐蔽的**：需要主角通过计算才能发现
3. **计算必须是可验证的**：读者可以自行验证主角的逻辑
4. **推翻的必须是基础设定**：不是推翻"一个规则细节"，而是推翻"游戏的最基本事实"

## 技法4：规则制定者的真实身份层——重构前提

最终层揭示：**规则制定者本人也是参与者**——他不是中立的裁判，而是游戏中最大的玩家。

### 4.1 主持人参与的设计方法

| 要素 | 设计 | 《十日终焉》案例 |
|:-----|:-----|:----------------|
| **隐藏身份** | 主持人的参与者身份被隐藏 | 配对机上有地羊的照片 |
| **隐藏扇子** | 主持人有自己的扇子和配对机会 | 地羊有160把扇子中的4把 |
| **杀人特权** | 主持人不受普通规则约束 | 地羊杀人无惩罚 |
| **利益动机** | 主持人在游戏中有自己的利益 | 地羊卖扇子赚钱 |

### 4.2 主持人参与的反转冲击

这个反转的叙事力量在于——它不是简单的"XX是个坏人"，而是对整个游戏认知的重构：
- 游戏不是"规则公平的竞技" → 是"猎手和猎物同场竞技"
- 主持人不是"裁判" → 是"隐藏的玩家"
- 规则不是"保护所有人的契约" → 是"主持人的狩猎工具"

### 4.3 主持人反转的写作模板

```markdown
主角发现了一个被所有人忽略的细节——
配对机上贴着主持人的照片。
主持人一直以"裁判"的身份站在一旁，遵守着规则。
但在所有参与者中——他的照片也在配对机上。
他也有扇子。他也有配对机会。
他不是裁判。他是玩家。
而且——他随时可以杀人而不受惩罚。
```

## 技法5：层间留白与节奏控制

每一层规则揭示之间需要有足够的**留白**——让读者（和角色）消化信息、重新理解游戏。

### 5.1 层间留白的设计

| 层次 | 揭示时间 | 层间留白 | 节奏 |
|:-----|:---------|:---------|:-----|
| 第一层（隐藏条件） | 第251章末尾 | 约6章 | 快节奏 |
| 第二层（数字谎言） | 第257章 | 约6章 | 中节奏 |
| 第三层（主持人参与） | 第258章 | 约1章 | 快节奏（密集揭示） |
| 第四层（终极对决） | 第260章 | 约2章 | 高潮节奏 |

### 5.2 留白期间的叙事

在留白期间不能"空转"——要让游戏在已有的规则认知下继续推进：
- 第252-256章（第一层和第二层之间）：贫富分化、欺诈博弈、暴力升级
- 第259章（第三层和第四层之间）：齐夏用新理解开展欺诈

这样每一层规则揭示之间都有**充实的情节发展**，读者不会感到"在等待答案"。

### 5.3 层间留白的写作原则

1. **揭示后立即行动**：每一层揭示后，主角立即用新认知行动
2. **不急于解释**：主角"明白了"但不一定立即解释给读者（留悬念）
3. **行动验证认知**：主角的行动让读者确信"主角真的理解了"
4. **下一层揭示深化**：下一层不是重复，而是推翻前一层的理解

## 组合使用指南

### 规则阶梯揭示章节的典型结构

```
游戏初期（第248-250章）：
  → 规则公布 + 参与者分工 + 欺诈开始
  → 读者以为自己理解了规则

第一层揭示（第251章）：
  → 隐藏条件出现：扇子要付费
  → 规则不是"完整的"——但大框架可信

第二层揭示（第257-258章）：
  → 数字谎言：参与者实际是51人
  → 连数据的"基本事实"都被推翻

第三层揭示（第258章）：
  → 主持人也是参与者
  → 整个游戏的前提被重构

终极对决（第260章）：
  → 主角与主持人正面对峙
  → 规则被彻底看穿后的终极博弈
```

### 与其他Skill的配合

- **`truth-layer-revelation`**（真相层次揭露）：本技法是真相层次揭露的"规则层面"应用——规则真相的层次结构与剧情真相一样需要层进式揭示
- **`rule-as-weapon`**（规则即武器）：在理解规则的深层真相后，主角可以利用规则作为武器
- **`meta-game-rule-trap`**（规则陷阱元博弈）：规则阶梯揭示揭示了主持人的元博弈层级——他不是在规则内博弈，而是用规则本身博弈
- **`death-game-narrative`**（死亡游戏叙事）：规则阶梯揭示为死亡游戏提供了"规则不可信"的新维度

## 注意事项

### 避免的陷阱

1. **不要一次揭示所有谎言**：如果所有谎言在第一章就被揭穿，后续章节就没有悬念了
2. **每个谎言之间要有因果**：不是随机谎言，而是每个新谎言都是对旧理解的深化
3. **后期谎言必须推翻前期理解**：最终层揭示必须让读者"重新理解整个游戏"——不是补充细节，而是推翻前提
4. **数学要在规则中有意义**：数字矛盾必须基于合理的游戏设计，不能为矛盾而矛盾

### 验证清单

- [ ] 第一层规则是否看起来合理完整？（不是一眼就看穿的谎言）
- [ ] 每个新层次的揭示是否"推翻但保留"前一层的合理性？
- [ ] 数字/数据的矛盾是否基于合理的游戏设计？
- [ ] 最后一层揭示是否重构了整个游戏的前提？
- [ ] 层间是否有足够的情节推进（不是纯等待答案）？
