## 二、新建Agent建议

### Agent 1：死亡游戏写手 (death-game-writer)

| 属性 | 内容 |
|:-----|:-----|
| **ID** | `death-game-writer` |
| **Emoji** | 🎲 |
| **部门** | 写作部门 |
| **职责** | 死亡游戏/无限流/规则怪谈类场景的机制设计、博弈叙事、心理对抗写作 |
| **调用方式** | `Agent(prompt=...)` |
| **核心能力** | ① 设计公平但可利用的游戏规则 ② 编写信息不对称博弈场景 ③ 构建多角色心理对抗 ④ 实现规则逐步揭示的节奏控制 |

**为什么需要**：
- 现有 `closed-space-suspense` 源自《夏日重现》的孤岛/影子设定，侧重恐怖氛围和信任崩塌，**不覆盖**死亡游戏的规则设计、博弈机制、信息不对称等核心技法
- 现有 `webnovel-suspense` 是通用悬疑指南，**不覆盖**游戏化叙事的特殊需求
- 《十日终焉》证明了死亡游戏是一种高度差异化的叙事类型，值得专项Agent

**职责边界**：
- 与 `identity-suspense-writer` 的区别：身份悬疑侧重"秘密身份"，死亡游戏侧重"规则博弈"
- 与 `closed-space-suspense` 的区别：封闭空间悬疑侧重恐怖氛围，死亡游戏侧重智力对抗
- 与 `plot-architect` 的区别：剧情架构师管宏观结构，死亡游戏写手管游戏单元内的微观博弈

## 三、新建Skill建议

### Skill 1：死亡游戏叙事 (death-game-narrative)

| 属性 | 内容 |
|:-----|:-----|
| **ID** | `death-game-narrative` |
| **来源** | 《十日终焉》第1-10章 |
| **目标目录** | `company/writing/skills/` |
| **适用场景** | 死亡游戏、无限流、规则怪谈、密室推理类章节 |

**核心技法**：

#### 技法1：信息不对称博弈设计
**来源**：《十日终焉》"说谎者"游戏
**步骤**：
1. 设计表面规则（所有人可见）
2. 设计隐藏规则（只有作者和后期角色知道）
3. 设计规则漏洞（可被聪明角色利用的灰色地带）
4. 设计规则延迟发布（分批释放规则，制造节奏波动）
5. 确保表面规则与隐藏规则之间存在逻辑必然性

**示例**：
> 表面规则：9人讲故事，找出1个说谎者
> 隐藏规则：所有人都是说谎者（都隐瞒了死亡事实）
> 规则漏洞：假名字也算说谎（姓名说谎）
> 延迟发布：身份牌机制在李尚武提出方案后才公布

#### 技法2：强制暂停的叙事功能
**来源**：《十日终焉》中场休息20分钟
**步骤**：
1. 在博弈高潮前设置强制暂停（中断紧张节奏）
2. 暂停期间释放"等待焦虑"（角色和读者都感到不安）
3. 暂停期间允许规则外的互动（串供、试探、信息交换）
4. 暂停结束后博弈层次升级（新规则/新信息/新联盟）

#### 技法3：元博弈思维
**来源**：《十日终焉》章律师质疑游戏本身
**步骤**：
1. 让角色质疑"游戏规则本身是否合理"（跳出框架思考）
2. 发现"赢面悖论"（如果某方赢面太大，游戏就失去意义）
3. 从元博弈推导出隐藏真相（游戏的目的不是表面所示）

#### 技法4：环境线索功能化
**来源**：《十日终焉》方格线→空气计算、座钟→时间控制
**步骤**：
1. 在场景建立时铺设"看似装饰"的元素（方格线、座钟、无门）
2. 在博弈过程中让角色/读者注意到这些元素
3. 在高潮揭示时将这些元素转化为解谜工具
4. 确保每个元素都有"双重功能"（氛围 + 线索）

### Skill 2：硬推理叙事 (hard-reasoning-narrative)

| 属性 | 内容 |
|:-----|:-----|
| **ID** | `hard-reasoning-narrative` |
| **来源** | 《十日终焉》第8-10章 |
| **目标目录** | `company/writing/skills/` |
| **适用场景** | 需要将数学/逻辑/科学推理融入叙事的章节 |

**核心技法**：

#### 技法1：结论前置推理法
**来源**：《十日终焉》齐夏先说"我们都死了"，再展示计算过程
**步骤**：
1. 先给出震撼结论（打破读者预期）
2. 再逐步展示推理过程（让读者同步验证）
3. 推理过程用精确数字/逻辑链（增强可信度）
4. 最终结论自然涌现（读者感到"原来如此"而非"强行解释"）

#### 技法2：数学即武器
**来源**：《十日终焉》空气计算推翻所有假设
**步骤**：
1. 在故事中埋设可量化的环境元素（空间尺寸、时间、数量）
2. 让角色在关键时刻进行精确计算
3. 计算结果与所有人的直觉判断矛盾
4. 用数学推理替代暴力/口才成为最可靠的说服手段

#### 技法3：专业语言的信息迷雾
**来源**：《十日终焉》赵医生的医学术语
**步骤**：
1. 让角色使用不可验证的专业术语
2. 读者和角色一样无法判断真伪
3. 专业知识成为博弈工具（信息护甲）
4. 在后续揭示中反转专业描述的含义

### Skill 3：群像博弈叙事 (ensemble-game-narrative)

| 属性 | 内容 |
|:-----|:-----|
| **ID** | `ensemble-game-narrative` |
| **来源** | 《十日终焉》第1-10章 |
| **目标目录** | `company/writing/skills/` |
| **适用场景** | 多角色在极端环境下的博弈/对抗/合作场景 |

**核心技法**：

#### 技法1：行动即人格
**来源**：《十日终焉》每个角色通过"危机反应"快速建立性格
**步骤**：
1. 每个角色在首次面对威胁时有独特反应（尖叫/沉默/观察/威胁）
2. 博弈策略反映核心人格（齐夏→数学计算、李尚武→组织联盟、赵医生→信息壁垒）
3. 角色间的互动暴露深层性格（质疑/合作/背叛）
4. 避免"作者解释"——让行动说话

#### 技法2：信息碎片拼图
**来源**：《十日终焉》每个角色的故事都是信息碎片
**步骤**：
1. 每个角色释放独立的信息碎片
2. 碎片之间存在矛盾/关联（两故事高度相似=巧合还是抄袭？）
3. 主角（和读者）需要拼出全貌
4. 矛盾不等于说谎——逻辑陷阱是核心难点

#### 技法3：利己的利他
**来源**：《十日终焉》齐夏救人动机是自保
**步骤**：
1. 主角的"高尚行为"有自私动机（"如果你们选错了，我也会死"）
2. 极端环境下道德判断不需要"高尚"理由
3. 利己动机驱动的利他行为更真实、更有力
4. 避免"圣母"人设——让动机灰度化

### Skill 4：真相层次揭露 (layered-truth-revelation)

| 属性 | 内容 |
|:-----|:-----|
| **ID** | `layered-truth-revelation` |
| **来源** | 《十日终焉》第8-10章 |
| **目标目录** | `company/writing/skills/` |
| **适用场景** | 多层反转、真相逐步揭露的高潮章节 |

**核心技法**：

#### 技法1：三层反转结构
**来源**：《十日终焉》谁说谎→为什么说谎→他们是什么
**步骤**：
1. 第一层反转：推翻表面问题（不是"谁在说谎"）
2. 第二层反转：更换问题类型（"为什么说谎"→"因为不愿承认死亡"）
3. 第三层反转：重构故事前提（"所有人都已死"→死后世界）
4. 每层反转都基于前一层的逻辑推导（不是强行反转）

#### 技法2：排比质疑的节奏感
**来源**：《十日终焉》齐夏逐一质疑每个角色
**步骤**：
1. 用相同句式结构质疑不同角色
2. 每次质疑后留出沉默/思考空间
3. 真相的重量感随质疑次数累积
4. 最后一个质疑引爆全局结论

## 四、更新Agent/Skill建议

### 4.1 更新 `closed-space-suspense` Skill

**当前状态**：源自《夏日重现》孤岛设定，覆盖信息隔离、社会结构、恐怖感营造
**缺口**：不覆盖死亡游戏的规则设计、博弈机制、信息不对称
**更新建议**：

在现有技法体系中新增：

#### 技法G：封闭空间的游戏化设计
**内容**：
- G1. 规则嵌入：封闭空间中嵌入可操作的游戏规则（不是纯恐怖，而是有"赢"的可能）
- G2. 信息不对称利用：利用封闭空间的信息隔离设计博弈
- G3. 环境功能化：封闭空间的每个元素都服务于游戏/解谜
- G4. 时间压力：座钟/倒计时等元素制造紧迫感

**来源补充**：《十日终焉》密室+死亡游戏

### 4.2 更新 `webnovel-suspense` Skill

**当前状态**：通用悬疑惊悚写作指南
**缺口**：不覆盖硬推理叙事、死亡游戏机制
**更新建议**：

新增章节：

#### 六、硬推理悬疑写作
- 数学/逻辑推理融入叙事的方法
- "结论前置"vs"过程展示"的节奏选择
- 专业语言作为信息迷雾

#### 七、死亡游戏/规则怪谈
- 游戏规则设计的公平性与可利用性

### 4.4 更新 `plot-rhythm` Skill

**当前状态**：10种剧情节奏技法，源自KonoSuba
**缺口**：缺少"游戏单元结构"和"信息不对称节奏"
**更新建议**：

新增技法：

#### §11 游戏单元结构
**来源**：《十日终焉》前10章构成完整"游戏单元"
**内容**：
```
开局设置（1章）→ 规则揭示（1章）→ 信息收集（3-5章）→ 高潮揭示（1章）→ 真相揭露（1章）→ 收束（1章）
```
- 每个阶段有明确的叙事功能
- 主题递进：生存恐惧→信任困境→存在危机→新起点

#### §12 信息不对称节奏
**来源**：《十日终焉》说谎者游戏
**内容**：
- 表面规则释放 → 角色利用规则 → 隐藏规则发现 → 元博弈质疑 → 真相揭露
- 每个阶段的信息释放量精确控制

## 七、总结

### 核心发现

1. **最大缺口**：死亡游戏/无限流叙事在系统中完全无覆盖。这是一种高价值、高差异化的叙事类型，值得专项投入。

2. **第二缺口**：硬推理叙事（数学/逻辑融入小说）完全无覆盖。《十日终焉》证明了这种写法的冲击力，但系统没有任何工具支持。

3. **已有覆盖的不足**：`closed-space-suspense` 和 `webnovel-suspense` 虽然有部分覆盖，但都源自不同的作品基因（夏日重现的恐怖悬疑），无法直接迁移死亡游戏的博弈机制。

4. **本能库空白**：《十日终焉》的分析产出了大量可沉淀的本能（预计50+条），但本能库中完全没有死亡游戏/硬推理相关的条目。

    ↓
