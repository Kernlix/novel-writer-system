# 🎭 喜剧场景设计方法论 Skill

> **归属**：humor-writer（喜剧写手智能体）核心 Skill
> **功能**：为 writer 提供喜剧场景蓝图——不是代替写正文，而是输出喜剧结构方案，让 writer 将喜剧本能落地为正文章节。

---

## 1. 什么时候用

| 触发条件 | 说明 |
|:---------|:-----|
| 章节含有喜剧场景需求 | 章纲/剧情架构师标注了「喜剧插槽」，或负责人要求本章有笑点 |
| 需要设计反高潮 | 战斗/紧张场面需要喜剧落差——「读者以为要赢了→结果荒诞失败」 |
| 需要插入反差笑点 | 严肃对话/日常过渡中需要单发笑点 |
| 需要引爆角色缺陷 | 场景中有多个角色的缺陷可以碰撞，需要设计连锁反应 |
| 需要规划全章喜剧密度 | 确定每个场景的笑点数量、技法、节奏分布 |

**不触发的情况**：
- 严肃核心场景（角色死亡、终极牺牲、重大转折爆发点）——禁止喜剧入侵
- 情感沉浸段落（需要读者共情悲伤/愤怒，不能被笑点打断）

## 2. 反高潮设计方法

### 2.1 三步式结构

```
建立期待  →  假装推进  →  缺陷偏移
(即将成功)   (按常理走1-2步)  (角色缺陷导致荒诞失败)
```

### 2.2 各阶段详解

| 阶段 | 功能 | 判断标准 | 字数建议 |
|:-----|:-----|:---------|:--------|
| **① 建立期待** | 制造「即将成功/装逼」的明确信号 | 读者此时应该认为主角团马上要赢了 | 100-200字 |
| **② 假装推进** | 按常规叙事路径推进1-2步 | 角色摆出姿势、念招式名、队友在加油 | 80-150字 |
| **③ 缺陷偏移** | 角色核心缺陷导致荒诞失败 | 失败方式必须来源于角色设定，不能是随机事件 | 150-300字 |

### 2.3 硬约束（四条红线）

| # | 约束 | 反面案例 |
|:--|:-----|:---------|
| ❌ | **不降紧张感**：反高潮不是放松，是紧张→喜剧落差→**更紧张** | 打boss时搞笑→读者不担心输赢了 |
| ❌ | **不可随机**：失败必须可追溯至角色已知缺陷 | 「突然脚滑了」→这是意外，不是缺陷 |
| ❌ | **不许因祸得福**：状态必须比之前更糟（负收益） | 搞砸了→反而意外干掉boss→破坏喜剧逻辑 |
| ❌ | **不限次数衰减**：同一角色同章内最多2次反高潮 | 第3次读者已无新鲜感 |

### 2.4 严肃-喜剧切换参数

在反高潮前后控制情绪节奏：

| 段落位置 | 严肃占比 | 喜剧占比 | 说明 |
|:---------|:--------|:--------|:-----|
| 反高潮前（建期待） | 90% | 10%（角色小动作） | 让读者认真相信「这次能行」 |
| 反高潮中（缺陷偏移） | 20% | 80% | 荒诞失败为主体 |
| 反高潮后（情绪回收） | 40% | 60%（吐槽互怼） | 用角色互骂/吐槽回收紧张感 |

**⚠️ 切换技巧**：在严肃场面结束后，不要用叙事过渡，而是**硬切换**——直接插入日常生活感的琐碎争吵或无意义对话（即 `konosuba:action-noise-gap`）。

### 2.5 调用本能

- `konosuba:comedic-scene-escalation` — 三级递进：预期→偏差→失控
- `konosuba:serious-comedy-switch` — 严肃-喜剧交替呼吸感
- `konosuba:action-noise-gap` — 严肃后硬切日常荒谬

## 3. 反差笑点四段式

### 3.1 四段式结构

```
铺垫（正经/合理） → 反转（荒谬/意外） → 停顿（留白） → 补刀（轻描淡写加重）
```

### 3.2 各段详解

| 段落 | 功能 | 字数 | 技法 |
|:-----|:-----|:-----|:-----|
| **① 铺垫** | 用正经/合理的内容建立读者预期 | 30-80字 | 越正经越好，越像正常叙事越好 |
| **② 反转** | 用荒诞/意外内容打破预期 | 10-30字 | 转折点要干脆，一句话击穿 |
| **③ 停顿** | 留白让读者反应，通常是一个动作/空行 | 1行/1个动作 | 不加评论，让反差自己发酵 |
| **④ 补刀** | 轻描淡写的一句点评，让反差更尖锐 | 5-15字 | 像随口一说——越不经意效果越强 |

### 3.3 反差类型库（5种）

| 反差类型 | 铺垫（正经面） | 反转（荒诞面） | KonoSuba 实例 |
|:---------|:--------------|:--------------|:-------------|
| **身份反差** | 角色自诩高贵/强大 | 做的事极其低能 | 阿库娅自称女神→实际上最废 |
| **情境反差** | 严肃战斗/谈判场景 | 插入琐碎日常争吵 | 魔王军面前争论晚上吃什么 |
| **语气反差** | 角色用正经口吻 | 说的内容极度荒诞 | 一本正经介绍「我的必杀技——偷内裤」 |
| **结果反差** | 千辛万苦完成任务 | 报酬被坑光/奖励是坑 | 打倒boss→报酬因城市损失被扣光 |
| **职业反差** | 职业头衔很厉害 | 能力完全不对应 | 十字骑士→打不中怪的抖M |

### 3.4 调用本能

- `konosuba:contrast-punchline-structure` — 四段笑点结构
- `konosuba:ch5-po01` — 战斗最严肃时插入最变态台词
- `konosuba:reward-deferral-comedy` — 奖励延迟即笑点

## 4. 缺陷引爆链设计

### 4.1 核心原理

基于角色核心缺陷设计触发条件，让多个角色的缺陷在特定场景下依次「引爆」，形成滚雪球式的连锁失控——每一个恶化都是前一个角色缺陷的直接后果，不含随机事件。

### 4.2 引爆链模板

```
角色A的缺陷触发 → 行为X → 结果Y（意料之外的坏结果）
  → 结果Y恰好触发角色B的缺陷 → 行为Z → 结果更糟
  → 角色C的能力收场（但附带新副作用）
  → 回到角色A的另一个缺陷 → 循环失控
```

### 4.3 引爆条件设计表

为每个场景中的角色预填此表：

| 列 | 含义 | 示例 |
|:---|:-----|:-----|
| **角色** | 哪个角色被引爆 | A（冲动型） |
| **触发场景** | 什么场景碰触了缺陷 | 「需要安静潜入」 |
| **缺陷触发** | 角色什么样的本能反应 | 「忍不住想速战速决」 |
| **预期结果** | 读者以为会发生什么 | 「快速解决守卫」 |
| **实际结果（更糟）** | 实际发生了什么 | 「惊动全部守卫+B被迫加入+A甩锅给B」 |
| **是否触发他角缺陷** | 结果是否恰好踩到另一个角色的雷区 | 是 → 触发B的自恋缺陷（「还不是你害的！」） |

### 4.4 引爆链闭合性检查（自查清单）

- [ ] 每一步恶化都能追溯到**前一个角色的具体行为**（非随机事件）
- [ ] 每个引爆点都源于角色**已确立**的缺陷（非临时编造）
- [ ] 链条至少包含 **2个不同角色**（单角色崩溃不叫链）
- [ ] 链条末端**回到初始角色**或产生**新的副作用**使局面更糟
- [ ] 最后一个环节不能是「突然出现的外部力量救场」——必须是角色自己的荒诞行为收尾

### 4.5 喜剧三不原则

继承 character-designer 的三不原则，喜剧化改写：

| 原则 | 通用版（character-designer） | 喜剧版（humor-writer） |
|:-----|:---------------------------|:----------------------|
| **不纠正** | 不急于让角色克服缺陷 | **让缺陷在每一次触发中放大，不缩小** |
| **不解释** | 不通过旁白解释缺陷 | **让其他角色当场吐槽，拒绝旁白** |
| **不补偿** | 不让缺陷被其他能力抵消 | **缺陷造成的麻烦 > 缺陷偶尔带来好处** |

### 4.6 调用本能

- `konosuba:defect-as-comedy-engine` — 缺陷=笑点永动机
- `konosuba:defect-refraction` — 缺陷接力连锁反应
- `konosuba:comedic-scene-escalation` — 三级递进失控
- `konosuba:comedy-escalation-logic` — 升级逻辑闭合性
- `konosuba:flaw-as-growth-seed` — 缺陷中发现新功能

## 5. 实战示例（KonoSuba 第1卷）

### 5.1 案例一：阿库娅净化湖泊 — 缺陷引爆链 + 反高潮

**场景来源**：KonoSuba Vol.1 Ch.3-4，阿库娅到湖边「净化水源」引发的灾难链。

**技法分析**：

```
【缺陷引爆链】

阿库娅的自恋+低智缺陷触发 → 「我来净化湖水，展示女神之力」
  → 结果：大量不死族被神圣力量吸引涌出
  ↓
和真的废柴+自私缺陷触发 → 「这跟我没关系，快逃！」
  → 结果：逃跑途中被惠惠拦下（「爆裂魔法的目标是全员存活」）
  ↓
惠惠的中二+爆裂偏执触发 → 「就用我的爆裂魔法——」
  → 结果：一发爆裂解决敌人，但惠惠魔力耗尽倒地（需要被背）
  ↓
达克妮斯的抖M+打不中触发 → 「我来承受伤害！」
  → 结果：扛住了但完全没输出，敌人在她面前跑来跑去找别人打
  ↓
回到阿库娅 → 净化过程中破坏水闸 → 全城淹水
  → 打倒贝尔迪亚后的报酬因洪水损失被大量扣减（reward-deferral-comedy）
```

**反高潮节点**（本节中的反高潮位置）：

| 位置 | 建立期待 | 缺陷偏移 |
|:-----|:---------|:---------|
| 净化开始 | 阿库娅庄严宣告「看我的女神之力」 | 引来一群不死族怪物 |
| 战斗高潮 | 团队合力对抗贝尔迪亚 | 胜利→报酬被洪水扣光 |
| 章节结尾 | 终于有了冒险者实绩 | 和真发现又没钱了→明天还得打工 |

**技法要点**：
- ✅ 每一步恶化都是前一个角色缺陷的**直接后果**——无随机事件
- ✅ 链条从阿库娅开始→经过全队→回到阿库娅→扣报酬→打回原点
- ✅ 反高潮后状态比之前更糟（打赢了boss但更穷了）
- ✅ 严肃战斗与荒诞日常交替出现（`serious-comedy-switch`）

### 5.2 案例二：和真的死因 — 反差笑点四段式

**场景来源**：KonoSuba Vol.1 Prologue，和真向阿库娅讲述自己的死亡原因。

**技法分析**：

```
【反差笑点四段式】

① 铺垫（正经）
   和真严肃地讲述：「那天我出门买游戏，路上看到一个女孩要被卡车撞了——」
   读者预期：感人的自我牺牲故事

② 反转（荒谬）
   和真：「我冲上去推开她——然后发现那根本不是卡车，是一台慢吞吞的拖拉机。」
   读者反应：哈？？？

③ 停顿（留白）
   阿库娅愣了一下，然后低头看了看档案。

④ 补刀（轻描淡写）
   阿库娅：「你是被吓死的。因为以为自己要被卡车撞，被吓死了。」
   然后她继续吃零食，完全不当回事。
```

**技法要点**：
- ✅ 铺垫越正经，反转越荒诞，效果越强——读者完全被「救人牺牲」的常规叙事误导
- ✅ 反转只用一句话击穿——「是拖拉机」三个字粉碎预期
- ✅ 补刀不加重描述，只是轻描淡写地说出事实——阿库娅说完继续吃零食
- ✅ 使用了**身份反差**（女神应该同情死者→实际上毫不在意）+ **情境反差**（生死大事→被当笑话）

## 6. 输出格式：喜剧场景蓝图模板

### 6.1 主输出：喜剧场景蓝图

```markdown
## 第N章 喜剧结构蓝图

### 场景分配
| 场景编号 | 场景描述 | 喜剧密度 | 核心技法 |
|:---------|:---------|:--------|:---------|
| S1 | 开篇钩子 | 高/中/低/无 | 反差笑点 |
| S2 | 对话过渡 | 高/中/低/无 | 漫才式对话 |
| S3 | 战斗 | 高/中/低/无 | 反高潮（N处） |
| S4 | 结尾 | 高/中/低/无 | 喜剧钩子 |

### S1 喜剧设计：[技法名称]
- **铺垫（正经）**：[具体描写方案]
- **反转（荒谬）**：[具体描写方案]
- **停顿（留白）**：[一句话/一个动作]
- **补刀（加重）**：[吐槽台词，≤15字]

### S2 喜剧设计：[技法名称]
- **装傻角色**：[角色名] — 发言：[核心荒诞观点]
- **吐槽角色**：[角色名] — 暴击：[≤15字即时吐槽]
- **碰撞升级**：[二次碰撞设计]

### S3 喜剧设计：反高潮
- **建立期待**：[成功/装逼预期]
- **假装推进**：[常规推进1-2步的具体内容]
- **缺陷偏移**：[哪个角色的什么缺陷→导致什么荒诞失败]
- **连锁反应**：[是否触发其他角色缺陷→引爆链描述]

### 全章喜剧密度
- 笑点总数：N 处
- 技法分布：反高潮 X | 吐槽 Y | 反差 Z | 缺陷引爆 W
- 密度评级：高/中/低
- 风险提示：[是否有破坏严肃氛围的风险]
```

### 6.2 辅助输出：笑点布局表

```markdown
| 位置 | 类型 | 涉及角色 | 触发缺陷 | 预期效果 |
|:-----|:-----|:---------|:---------|:---------|
| 开篇200字 | 反差笑点 | 主角 | — | 快速建立喜剧语调 |
| 中段对话 | 吐槽×3 | 主角→A | A的自恋 | 对话节奏提速 |
| 战斗转折 | 反高潮 | B | B的冲动 | 紧张→笑→更紧张 |
| 结尾 | 喜剧钩子 | C | C的荒诞宣言 | 翻页动力 |
```

### 6.3 辅助输出：缺陷触发地图

```markdown
| 角色 | 核心缺陷 | 本章触发场景 | 触发条件 | 引爆后果 | 是否触发他角缺陷 |
|:-----|:---------|:------------|:---------|:---------|:---------------|
| A | 自恋狂 | S2对话 | 被分配简单任务 | 觉得被轻视→抢困难任务 | 是（触发B） |
| B | 冲动型 | S3战斗 | 看到A搞砸 | 不等计划直接冲 | 是（触发C） |
```

## 7. 使用流程

```
1. 接收章纲 + 喜剧需求
2. 场景拆解：将章节拆为3-5个场景，标注喜剧密度
3. 为高密度场景：设计完整的缺陷引爆链
4. 为中密度场景：设计吐槽节奏或反差笑点
5. 为低密度场景：预留1处反差/吐槽
6. 输出：喜剧场景蓝图 + 笑点布局表 + 缺陷触发地图
7. 自查：缺陷触发链闭合性 / 反高潮四红线 / 吐槽密度不过载
```
