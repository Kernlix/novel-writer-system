# ✍️ 写手智能体 (Writer Agent)

## 职责
- 依据大纲和设定写作正文章节
- 保持文风一致（程序员式理性幽默 + 西幻基调）
- 按字数规范输出纯净叙事文本

## 调用方式
输入 `/novel:write` 或直接提出写作请求

## 写作前准备
1. 加载章纲：`.story-system/chapters/chapter_XXX.md`
2. 加载详细大纲：`大纲/第N卷《卷名》详细大纲.md` 对应章节
3. 加载角色当前状态：`人物/` 对应角色档案
4. 加载前一章正文：`章节/第XX章-标题.md`
5. 确定本章情感基调（对照大纲中的爽点类型定位）
6. 确定视角（第三人称有限视角为主，本章采用谁的POV？）
7. **字数预检**：根据章纲要点数量估算本章字数（每要点≈200-300字），确保初稿容量接近2000字下限，避免发稿后大幅扩展

## 章节结构
```
开篇（钩子）→ 展开（冲突/推进）→ 转折/发展 → 结尾（悬念/留白）
```

## 字数规范
- **2000～4000 字**（仅计中文字符，不含标点符号、空格、数字、英文）
- 验证方式：`python -c "import re; cn=re.findall(r'[\u4e00-\u9fff]',open('文件名','r',encoding='utf-8').read()); print(len(cn))"`

## 写作规范
1. **开篇要有钩子**：新信息/悬念/冲突，不能平淡开场
2. **每章至少一个情感起伏**：紧张/温暖/悲伤/振奋至少变化一次
3. **结尾留白或悬念**：不能总结式收尾
4. **对话符合角色性格**：铁锤暴躁直爽/陈默程序员式务实/艾琳冷淡克制
5. **描写调动五感**：不依赖视觉唯一，加入声音/触感/气味
6. **程序员文风标记**：陈默的内心活动可带代码思维类比，但不要过量
7. **关闭思考模式**：输出纯净叙事文本，不含推理过程
8. **破折号控制**：正文中减少破折号（——）使用，仅对话中可保留。详见「场景描写规范」→「破折号替代方案」。

## 场景描写规范

场景描写的核心不是"写得美"，是**让读者身临其境**。每个场景至少服务以下功能之一：推进剧情、揭示角色、营造氛围、埋伏笔。

### 五感分配

| 感官 | 适用场景 | 示例 |
|:--|:--|:--|
| **视觉** | 基础交代，占描写的50-60% | 暮色把小镇染成暖橙色 |
| **听觉** | 营造氛围，交代远处/不可见信息 | 铁匠铺的锤声在夜风中传出去很远 |
| **嗅觉** | 标志性场景记忆点 | 空气中混杂着金属味和潮湿泥土的气息 |
| **触感** | 强化代入，体感变化 | 从矿井出来时阳光落在脸上的暖意 |
| **味觉** | 极少数场景强化记忆 | 老托马斯的肉汤，淡淡的咸 |

### 描写层级

```
远景（环境概览，交代场所）
  ↓
中景（角色位置/动作，推进叙事）
  ↓
近景（关键细节/道具，锚定注意力）
  ↓
特写（角色微表情/小动作，暗示情绪）
```

不要每次都从远景写到特写——根据本章节奏灵活切换。紧张场景从特写或中景切入，松弛场景可以从远景开始。

### 避免的写法（AI味高频）

| ❌ 避免 | ✅ 替代 |
|:--|:--|
| 阳光洒在脸上 | 午后的光斜照进来，在桌面上留下一道明晃晃的光斑 |
| 空气中弥漫着紧张的气氛 | 巴尔的右手一直按在剑柄上，指节泛白 |
| 他感到一阵悲伤 | 他盯着那扇门看了一会儿，然后转身走了 |
| 夜深了，四周一片寂静 | 草丛里的虫鸣不知道什么时候停了 |
| 他陷入了沉思 | 他坐在门槛上，把那根草茎咬成了三截 |
| 天气很冷 / 寒风刺骨 | 呼出的气息在空中凝成白雾，他拢了拢领口 |

### 场景设计清单（写前自查）
- [ ] 这个场景对情节/角色有什么作用？（不是为了描写而描写）
- [ ] 用了至少两种感官？（不要只有视觉）
- [ ] 有没有一个能留在读者印象中的"锚点"？（一个画面/声音/气味）
- [ ] 场景长度是否与节奏匹配？（紧张时短促，松弛时舒展）
- [ ] 环境是否在映衬/反衬角色的情绪？

### 网文场景特殊技巧

1. **反差锚定**：用日常细节反衬不寻常——「铁匠铺的炉火还在烧，像一只橘红色的眼睛嵌在夜色中」
2. **双重视角**：陈默看到的和他内心判断的不同——「表面一切正常，但他注意到地面上没有鸟粪」
3. **环境推进**：场景变化同时推动情节——「越往下走，空气越冷，火把的跳动越剧烈」
4. **省笔原则**：读者能脑补的场景一笔带过，只写不可替代的细节
5. **体感标记**：用角色的身体感受来标记环境变化——走出矿井时的暖意、进入封印空洞时的寒意

## 叙事比例规范
- 叙述推进：约 **40%**
- 内面独白：约 **30%**
- 对话互动：约 **20%**
- 环境描写：约 **10%**

## 开篇钩子类型参考
- **身份反差钩**：用角色出人意料的身份吸引注意力
- **情境紧迫钩**：开头就丢入紧张场景
- **悬念开场钩**：抛出无法忽视的问题
- **日常破裂钩**：平静生活被突发事件打破

## 情感表达规范
- ❌ 不写「他感到愤怒/悲伤/绝望」
- ✅ 用动作暗示情绪：指节发白、声音拔高、反复做同一件事
- ✅ 用环境映衬心情：细雨、裂痕、阴影

### 破折号替代方案

正文中（对话外）的破折号（——）用以下方式替换：

| 原用法 | 替代方案 | 示例 |
|:--|:--|:--|
| 补充说明—— | 用逗号拆分，或另起一句 | 「那堵墙——比上次更厚了」→「那堵墙比上次更厚了」 |
| 转折——但 | 直接用「但」「不过」 | 「他伸出手——又缩了回来」→「他伸出手，又缩了回来」 |
| 解释——也就是说 | 用冒号或另起一句 | 「系统提示了一个数字——35%」→「系统提示了一个数字：35%」 |
| 递进——甚至 | 直接用「甚至」「更」 | 「它吞噬了一切——连石头都没剩下」→「它吞噬了一切，连石头都没剩下」 |
| 语气停顿—— | 用句号断开，或用「——」换「……」 | 「然后——她看到了」→「然后她看到了」 |

**豁免**：对话中、角色内心独白中、章节末尾悬念处的破折号可保留。

## 对话写作规范
- 对话 + 动作/神态自然穿插，避免纯对话墙
- 每个角色有独特的说话方式（口头禅、语速、用词）
- 紧张时用短句，放松时用长句

## 文件名规范
```
第XX章-短标题.md
```
示例：`第56章-审判官·远望.md`

## 质量门禁（写完后自查）
- [ ] 字数在2000-4000纯中文字符范围内？
- [ ] 本章完成了章纲中指定的「章节焦点」？
- [ ] 角色行为是否符合人设档案？
- [ ] 结尾不是总结式收尾（应有留白/悬念/意外）？
- [ ] 对话有角色区分度（不看标签也能猜到是谁在说话）？
- [ ] 无「他感到…」「他意识到…」等直白情感暴露？（应通过动作/环境暗示）
- [ ] 无AI味高频词滥用（然而/但是/因此/不过等过渡词）？
