# 环形信息链谋杀设计 Skill

> **核心来源**：《十日终焉》第302-310章——地蛇「少数与多数」游戏
> **提炼方法**：从死亡游戏机制、心理博弈维度中提取可复用叙事技法
> **适用场景**：死亡游戏、信息链博弈、传话类游戏、群决策谋杀、封闭空间博弈章节
> **与 `rule-as-weapon` 的区别**：`rule-as-weapon` 聚焦「利用规则漏洞」的战术层面；本Skill 聚焦「信息传递链本身作为杀人武器」的**设计层面**——不是利用规则的漏洞，而是利用信息传播固有的不可靠性
> **与 `meta-game-rule-trap` 的区别**：元博弈聚焦「让系统本身制裁对手」；本Skill 聚焦「信息在传递中必然被扭曲」这一本质属性来设计谋杀

---

## 核心原理

**环形信息链谋杀**的核心设计思路是：参与者在物理上隔离，唯一的信息通道是一条封闭的电话/传话环链。信息（问题/消息）沿环链传递，每个节点都可以修改它，且游戏规则**不惩罚**这种修改。最终经过多轮传递后的集体决策触发致命的物理机关。

关键创新点在于：**武器不是规则的漏洞，而是信息传递的不可靠性本身**。

### 与普通「传话游戏」的区别

| 维度 | 普通传话游戏 | 环形信息链谋杀 |
|:-----|:------------|:--------------|
| 传话目的 | 娱乐/考验记忆力 | 决定生死 |
| 信息修改 | 无意的失真 | 有意的恶意篡改 |
| 传递拓扑 | 线性（头到尾） | 环形（可循环） |
| 决策机制 | 最后一人说出结果 | 全体投票/多数决触发机关 |
| 死亡关联 | 无 | 投票结果直接关联机关触发 |

## 技法1：信息链拓扑设计

### 1.1 环形拓扑（本批案例）

12个参与者呈环形排列，每人一个独立隔音房间。电话系统只能单向传递给**左手边的下一个房间**。关键设计要点：

- **单向传话**：电话只能打给左手边，无法反向联系或跨级联系——制造信息不对称
- **隔音+铁墙**：参与者之间无法喊话交流，只能依赖电话链——强化信息链的排他性
- **自动转接**：如果某人不接电话（已死或不想接），系统自动转接到下一个活人——死亡不破坏信息链

### 1.2 信息链长度的数学设计

信息链的参与者数量（n）决定了叙事空间：

| n值 | 叙事效果 | 风险 |
|:----|:---------|:-----|
| n=3-5 | 信息传递快，读者易跟踪 | 篡改空间小，过早暴露凶手 |
| **n=10-13** | **篡改空间大，悬念持续** | **（推荐）读者能跟踪但不易预测** |
| n>15 | 悬念感强 | 读者丢失跟踪，参与感下降 |

**十二人是地蛇游戏的优选数字**：一年有12个月，钟表有12小时——12是天然的文化记忆锚点。同时12个位置的信息链长度足够让读者感到复杂但又不完全迷失。

### 1.3 环形等价性

在环形拓扑中，远端的参与者实际上是「近邻」。例如在12人环中，-8的位置等价于+4。这个特性可以制造「看似无关实则致命」的悬念——表面远离你的人，可能在环中就在你身边。

## 技法2：信息篡改的多层次设计

### 2.1 篡改动机设计

参与者的修改动机通常有四种：

| 类型 | 动机 | 叙事功能 |
|:-----|:-----|:---------|
| **无意修改** | 紧张/误听/遗忘 | 制造第一次「意外死亡」，游戏真实面目暴露 |
| **自保修改** | 被铁球瞄准时修改问题自救 | 展示规则的可操控性，揭示「第一人改题权」 |
| **恶意杀人** | 减少竞争者/私人恩怨 | 将游戏推向主动谋杀的道德深渊 |
| **操控型修改** | 操纵群体选择达到战略目标 | 主角从被动参与者到主动操盘手的转变点 |

### 2.2 问题嵌套结构（亮点技法）

核心技法：**三层问题嵌套**——同一道题目在传递过程中被多次修改，形成信息博弈的「元游戏」层次。

```
第一层：原始问题
  → 「要落下吗」（地蛇设计的原始致命问题）

第二层：被自救者修改
  → 「一周有六天吗」（被铁球瞄准的第一人修改，引导大家选「否」来自救）

第三层：被操纵者再修改
  → 「最终瓜分的筹码要翻倍吗」（陈俊南覆盖，利用贪婪驱动所有人选「是」）
```

**写作要点**：
- 三层修改要有**递进性**——每层修改都基于对上一层的理解和反应
- 最后一层的修改必须是**反直觉的逆转**——不是让读者猜到「选否」，而是用完全不同的框架（贪婪）覆盖原有的框架（生死）
- 读者应该在事后能回溯每一层修改——增加「原来如此」的阅读满足感

## 技法3：界面语义重构（范式转换）

### 3.1 核心概念

游戏的视觉/交互界面（如两个按钮，标注「是/否」或「对/错」）的真实语义可能完全不同于其表面标签。主角通过**实验方法**发现界面语义的真相：

1. **假设**：按钮不是「对/错」，而是「左/右」
2. **验证**：故意选「错」（选「是」当自己是女性）——但未受惩罚，验证假设
3. **建模**：将答案映射为坐标系中的方向指令（是=-1，否=+1）
4. **应用**：利用建模预测铁球移动轨迹并主动操控

### 3.2 日常经验作为触发点

范式转换的灵感应来自角色的**日常/童年经验**——这增加了角色的亲切感，同时避免纯逻辑推演的枯燥。

> 案例：陈俊南从小霸王游戏机触发联想——左右方向按钮控制移动。这个接地气的类比让复杂的逻辑推理变得「人人都能理解」。

### 3.3 范式转换的叙事结构

```
① 反常观察 → 角色发现某个行为未产生预期惩罚
② 假设形成 → 「也许这个根本不是对错判断」
③ 实验验证 → 角色主动测试规则边界
④ 新模型建立 → 「是/否 = 左/右」
⑤ 新模型应用 → 利用新理解预测/操控游戏
```

## 技法4：预告-执行闭环

### 4.1 核心机制

角色做出一个**具体、可验证的死亡预告**，然后在后续章节中实际执行。读者在等待验证的回合中持续保持紧张和期待。

### 4.2 结构模板

```
发布预告 → [悬念章节1] → [悬念章节2] → 预告兑现

发布预告：「三个回合之内，我会让其中一个参与者惨死」
第1回合：一切正常，读者开始怀疑
第2回合：局势微妙变化，读者感到不安
第3回合：预告兑现，冲击力加倍
```

### 4.3 写作要点

- **预告必须具体**（谁/什么时候/以什么方式）——越具体越有冲击力
- **预告必须有挑战性**——读者不相信角色能做到
- **验证过程不能太快**——需要足够的回合数让读者产生期待，也需要让读者怀疑预告是否会成真
- **兑现必须无法被质疑是巧合**——必须是主角计划的一部分

## 技法5：杀人的「物化」处理

### 5.1 核心机制

当主角需要主动杀人时，用**几何代号**（正方形、三角、圆圈）取代对手的姓名/样貌——既是认知负担的降低（读者不用记很多名字），也是角色情感疏离的隐喻（杀人者不把对手当人）。

### 5.2 叙事功能

- **降低道德感**：代号化让杀人行为变得更「抽象」，减少读者对杀人者的道德谴责
- **增强脑力感**：几何代号暗示主角像下棋一样运筹帷幄——对手是棋子而非活人
- **悬念保留**：不给死者名字/样貌——避免读者对死者产生共情干扰主线情绪
- **死者命名权的夺取**：死者以主角赋予的代号被记忆——主角拥有「定义」牺牲者的叙事权力

## 综合叙事弧

一个完整的环形信息链谋杀游戏的叙事应该遵循以下阶段：

```
第一阶段：【规则展示】— 介绍信息链拓扑、传递规则、投票机制
    → 读者了解游戏怎么玩，但不知道死亡如何触发

第二阶段：【第一次意外】— 有人无意中发现信息可以修改或扭曲
    → 紧张感引入：「原来信息会失真！」

第三阶段：【第一次死亡】— 某人的恶意修改导致铁球/机关落下
    → 游戏真实面目暴露：「这不是游戏，这是谋杀」

第四阶段：【机制推理】— 主角通过实验和推理理解完整的规则模型
    → 从「被动参与」到「主动理解」

第五阶段：【范式转换】— 主角重新定义游戏的基本交互语义
    → 「对/错」变成「左/右」——游戏本质被改写

第六阶段：【主动操控】— 主角利用信息链执行自己的战略
    → 做出预告→执行杀人→证明自己理解正确

第七阶段：【更大布局】— 游戏内的杀人服务于游戏外的战略目标
    → 杀人不是目的，而是手段
```
