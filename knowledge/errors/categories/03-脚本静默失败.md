# 脚本静默失败

## 定义
批量操作（cp/mv/批量替换）失败但没报错，静默丢失数据

## 典型场景
- 正则不匹配导致记录消失
- cp没检查返回值
- 替换没做前后计数对比

## 自动检查
```bash
BEFORE=$(grep -c '模式' 文件); # 执行修改; AFTER=$(grep -c '模式' 文件); [ "$BEFORE" = "$AFTER" ] || echo '❌ 数据丢失'
```

## 历史同类型错误
- 2026-07-11_batch-silent

## 对策
1. 发生该类错误时，立即记录到 `knowledge/errors/entries/`
2. 更新 `root-causes.json` 计数
3. 更新 `README.md` 速查表
4. 将自动检查脚本写入 `pre-commit-check.sh`

## 系统教训
- 该类错误已发生 4 次，证明光靠记忆不可靠
- 必须通过自动化脚本+pre-commit检查来阻断
