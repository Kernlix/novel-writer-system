# 技法三：经济系统的自洽循环（负债叙事学）

> 不是章节推动角色，是账单推动角色。

## 原文/原片段

素晴的经济系统是串联剧情的隐形引擎。和真的经济轨迹可以概括为一条不断循环的锁链：

### 经济循环链

```
欠债（城堡修复费/破坏公物赔款）
  → 不得已接高级任务
    → 任务中搞出更大破坏
      → 赔款 > 报酬
        → 负债增加
          → 需要接更危险的任务
            → 循环
```

### 具体例证

**（1）开局负债**
和真转生到异世界，阿克娅作为新手礼包跟来。一开始没欠债，但随着"女神要生活""要装备""要训练"，很快花光了新手资金。

**（2）城堡事件**
第一次大事件：击退Dullahan后，城堡受损严重。虽然是打败敌人的副产物，但和真小队被要求赔偿。这里埋下关键逻辑：**保护城市≠不需要赔偿**。这不是NPC抠门，这是讽刺了"英雄不需要为破坏负责"的常见设定。

**（3）公务任务**
为了还债，不得不接魔王军相关的高危任务。正常冒险者公会体系中，低级冒险者不能接高级任务——但因为"这是以魔王军为对象的公务"，跳过了等级限制。经济压力让规则为叙事破例，但破例本身有合理理由（公务性质）。

**（4）惠惠的爆裂坑**
日常开支中，惠惠每天一次的爆裂魔法不仅是战斗技能，也是经济负担——炸完地形的修复费计入小队账目。技能设定与经济系统打通了！

**（5）阿克娅的败家**
阿克娅赚多少花多少（宴会、奢侈品、被骗钱），经济缺口永远存在。角色的性格缺陷在经济行为中自然体现，而不是"突然需要钱了"。

### 数字细节（仅第一卷）

| 事件 | 收支 | 说明 |
|------|------|------|
| 新手装备 | -5万Eris | 基本装备费用 |
| Dullahan战 | +3万报酬 | 击败强敌 |
| 城堡维修 | -15万赔款 | 战斗中破坏公物 |
| 后续任务 | +酬劳但不足 | 日常任务杯水车薪 |
| 净结果 | 持续负债 | 越打越穷 |

## 技法分析

### 核心洞见

素晴的经济系统揭示了一个关键原理：**经济系统不是背景板，是叙事引擎。**

在很多作品中，角色接冒险任务的动机是"因为冒险者要做好事"或"因为剧情需要"。而在素晴中，动机是赤裸裸的：**没钱了。**

这个看似简单的设定完成了三件事：

### 1. 提供不可抗的任务驱动力
当主角想摆烂的时候，账单不允许。创作者不需要绞尽脑汁安排"意外事件"推主角出门，只需要让钱包见底。

### 2. 在经济行为中完成角色塑造
- 阿克娅→赚了钱就挥霍→浪费的性格
- 惠惠→炸地形增加赔款→明知故犯的任性
- 达克妮丝→故意接高风险低报酬的任务→受虐癖
- 和真→试图省钱→各种失败操作→贪婪而倒霉

每个角色的经济行为都反映其核心性格。钱是性格的试纸。

### 3. 自洽循环降低读者质疑
读者不会问"他们为什么要接这么危险的任务？"——因为前一章的结局已经写了"他们欠了一屁股债"。因果链清晰，读者接受。

## 可复用场景

### 场景一：冒险/任务制作品

设计经济循环的基本原则：
```
奢侈品诱惑（可买可不买的好东西）
  → 角色忍不住花钱
    → 钱不够了
      → 必须接任务
        → 任务报酬 - 任务装备消耗 - 日常花销 = 微薄结余
          → 循环
```

**质检要点**：
- 不可让角色"做完一个大任务后暴富"
- 必须在报酬到手前就安排好花钱的出口
- 每一笔重大收入都对应一个更大的支出预期

### 场景二：种田/建设类作品

经济循环反向设计：
```
前期：负债开局（租地/贷款买种子）
  → 第一季收成还一部分贷款
    → 需要升级设备→再贷款
      → 扩大生产→还更多→贷更多
        → 持续的压力
```

注意：种田类更注重"逐渐变好"的体验，但过程中的压力不能消失。

### 场景三：政治/权谋类作品

经济循环升维：
- 不是个人的钱，是领地/国家的财政
- 军费开支 → 税收不足 → 对外战争抢资源 → 战争消耗更多军费 → 经济压力升级
- 角色面临的选择：增税（民心下降）vs 减少开支（防御变弱）

## 自我提升方案

### 对照分析

当前 `resource-economy.md` 的检查清单侧重于数学合理性的静态核算（人口匹配、战力数值等），缺少对**经济循环的动态叙事效能**的评估。数学上自洽的经济系统不一定是好系统——它还需要驱动叙事。

### 优化措施

1. 在 `resource-economy.md` 中新增"经济循环的叙事驱动力"检查清单（见 upgrade 字段）
2. 在 `setting-qa-agent.md` 的核心框架中新增"资源循环叙事效能"维度
3. 设计"经济循环压力测试"：模拟删除经济系统后，看剧情推进是否受影响
4. 在 QA 报告中增加"经济循环闭环完整性"指标：哪些环节断裂了？

### 应用承诺

当我审查资源经济设定时，我不仅查看数字合理性，还会追问：
- 角色的经济压力是持续的还是间歇性的？如果是间歇性的，间隙期是否缺乏驱动力？
- 经济系统是否与角色性格绑定？（挥霍型角色必须安排花钱渠道，守财型角色必须让他被迫花钱）
- 最大一笔债务的催收期限是什么？→ 这通常就是下一个大事件的触发点
