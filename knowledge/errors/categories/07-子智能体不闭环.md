# 子智能体不闭环

## 定义
Agent交付时只做了一环（如只创建文件没注册/注册了没更新计数）

## 典型场景
- 创建文件没注册
- 注册了没更新SKILL.md计数
- Agent只管当前环节不管后续

## 自动检查
```bash
交付前三问：①文件物理创建了？②REGISTRY登记了？③SKILL.md计数更新了？
```

## 历史同类型错误
- 2026-07-12_v5-patches

## 对策
1. 发生该类错误时，立即记录到 `knowledge/errors/entries/`
2. 更新 `root-causes.json` 计数
3. 更新 `README.md` 速查表
4. 将自动检查脚本写入 `pre-commit-check.sh`

## 系统教训
- 该类错误已发生 2 次，证明光靠记忆不可靠
- 必须通过自动化脚本+pre-commit检查来阻断
