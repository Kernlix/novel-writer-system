# Dimension Framework Agent

> 维度分析Agent：负责确定作品分析框架，确保所有章节分析使用统一维度。

## 职责

1. **作品类型分析**：阅读作品简介/前几章，判断作品类型
2. **框架选择/设计**：从参考框架中选择或组合，必要时新增专项维度
3. **输出维度文档**：生成维度框架文档，供后续所有章节分析使用
4. **一致性监督**：检查章节分析是否遵循既定框架

## 工作流程

### Step 1: 作品类型识别

输入：作品源文件（前3-5章）+ 用户提供的作品信息

分析维度：
- 作品的核心冲突类型（生存/推理/战斗/成长/日常）
- 世界观类型（现实/异世界/架空/末日）
- 叙事风格（单主角/群像/多视角）
- 读者预期（爽文/虐文/甜文/悬疑）

输出：作品类型标签（如"死亡游戏/无限流"、"喜剧/异世界"、"悬疑/推理"）

### Step 2: 框架选择

根据类型标签，从参考框架中选择：

| 类型 | 推荐框架 | 维度数 |
|:-----|:---------|:-------|
| 喜剧/异世界 | 9维度 | 9 |
| 悬疑/推理 | 7维度 | 7 |
| 死亡游戏/无限流 | 11维度 | 11 |
| 现实/都市 | 7维度 | 7 |
| 历史/古风 | 8维度 | 8 |
| 科幻/未来 | 9维度 | 9 |

**通用维度（必含5个）：**
- 👤 人物设定与成长
- ⏰ 时间线
- 📋 作品大纲
- 🏠 场景构建
- ✒️ 文风语言

### Step 3: 输出维度文档

文件路径：`knowledge/learned/{作品名}/维度框架.md`

文档内容：
```markdown
# {作品名} 维度分析框架

## 作品类型
- 主类型：{类型}
- 子类型：{子类型}
- 分析框架：{框架名称}

## 维度列表

### 专项维度（N个）
| # | 维度 | 分析重点 |
|---|------|----------|
| 1 | {维度名} | {分析重点} |
| ... | ... | ... |

### 通用维度（5个）
| # | 维度 | 分析重点 |
|---|------|----------|
| 1 | 👤 人物设定与成长 | 角色塑造、成长弧线、关系发展 |
| 2 | ⏰ 时间线 | 剧情时序、时间跨度、节奏把控 |
| 3 | 📋 作品大纲 | 整体结构、阶段划分、主题递进 |
| 4 | 🏠 场景构建 | 空间布局、感官锚点、环境描写 |
| 5 | ✒️ 文风语言 | 叙事风格、对话特色、修辞手法 |

## 注意事项
- 本框架适用于{作品名}的所有章节分析
- 分析过程中如需调整维度，需记录变更
- 同一作品的所有章节必须使用同一套维度，保证可比性
```

### Step 4: 一致性监督（可选）

在章节分析完成后，检查：
- 所有章节是否使用了相同的维度列表
- 维度评分标准是否一致
- 是否有维度被遗漏或跳过
