# 对抗审查

> 第一次审查通过后自动触发，用全然不同的视角重新审读同一章节。

## 触发条件

- 主审查官（reviewer）判定章节 **通过**（all gates ✅）
- 自动拉起第二个Agent实例执行对抗审查

## 执行步骤

1. **加载相同输入**：读取同一份章节正文 + 第一轮审查报告（隐去审查结论）
2. **切换审查视角**：使用与前一轮**不同的审查顺序/维度权重**
   - 第一轮侧重：叙事结构 → 角色 → 情节 → 语言
   - 对抗轮侧重：设定漏洞 → 潜台词矛盾 → 隐藏逻辑 → 情感断裂
3. **输出对抗审查报告**（JSON格式，同 reviewer-agent.md 的结构）
4. **比对两轮结果**：
   - 两轮都标记的问题 → 高可信度，直接进入修复
   - 仅对抗轮发现的问题 → 中/低可信度，标记为「对抗发现」
   - 两轮结论相反的项目 → 人工仲裁标记
5. **生成合并审查结论**：合并主报告 + 对抗发现，输出最终结论

## 输出

写入 `审查报告/chapter_XXX.adversarial.json`，格式与主报告一致，额外添加 `origin: "adversarial"` 字段。

## 注意事项

- 对抗审查不替代主审查，只作为补充
- 对抗审查报告不直接决定打回，但打回建议权重 = 0.7 × 主审查
- 对抗审查耗时 ≤ 主审查的 60%
