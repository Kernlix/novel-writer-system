# 声明没实现

## 定义
口头/文档说新增了功能，但实际文件不存在或流程不通

## 典型场景
- 新增hook但不创建文件
- 注册agent但不写prompt
- 说支持但不写实现

## 自动检查
```bash
[ -f "$声明的文件" ] || echo '❌ 声明了不存在'
```

## 对策
1. 发生该类错误时，立即记录到 `knowledge/errors/entries/`
2. 更新 `root-causes.json` 计数
3. 更新 `README.md` 速查表
4. 将自动检查脚本写入 `pre-commit-check.sh`
