# 《十日终焉》标准学习任务模板

> 每批学习必须包含以下3个阶段，不可跳过。

## 阶段1：并行分析（3个代理）

### 代理分配
- 代理A：第N-N+3章（4章）
- 代理B：第N+4-N+6章（3章）
- 代理C：第N+7-N+9章（3章）

### 每个代理的任务
1. 从源文件提取原文（用 `grep -n` 定位章节行号，`sed -n` 提取）
2. 对每章进行11维度分析（死亡游戏/无限流框架）
3. 输出独立分析文件到 `knowledge/learned/十日终焉/`
4. 文件命名：`第{N}章-{章节标题}.md`

### 11维度框架
🎲死亡游戏机制 ⚡超自然能力体系 🧠心理博弈 💀人性探讨 🔮悬念与伏笔 🌍世界观架构 👤人物设定与成长 ⏰时间线 📋作品大纲 🏠场景构建 ✒️文风语言

## 阶段2：汇总学习（1个代理）

**等待阶段1全部完成后执行**

### 任务
1. 读取本批10个章节分析文件
2. 撰写综合分析报告（`第{N1}-{N2}章-综合分析.md`）
3. **检查是否有新技法**（前200章未覆盖的写作技法）
4. 如有新技法，列出清单供阶段3使用

### 综合分析内容
- 本批概览（10句话总结）
- 核心发现（按维度分类）
- 技法归纳（可迁移的写作技法）
- 与前200章的关联
- 新增技法清单（如有）

## 阶段3：知识管线（必做⚠️）

**等待阶段2完成后执行**

### 3.1 缺口分析
- 检查本批是否有新的写作技法
- 对比已有Skill列表（`company/REGISTRY.md` 第51行）
- 判断：新建Skill / 合并到已有Skill / 无需新增

### 3.2 Skill创建（如需要）
- 新Skill文件：`company/writing/skills/{name}.md`
- 格式：frontmatter（id/name/skill/agent/description/created/source）+ 具体技法 + 配合指南

### 3.3 注册（三文件同步）
1. `company/REGISTRY.md` — 第51行Skills列表追加
2. `company/writing/writer-agent.md` — 可用技能表新增
3. `company/writing/skill-matcher-agent.md` — 章节类型映射表新增

### 3.4 更新学习计划
- `knowledge/learned/十日终焉/学习计划.md` — 批次状态 ⏳→✅

### 3.5 验证
- 检查所有文件是否存在
- 运行 `bash scripts/pre-commit-check.sh`（如有）

## 输出给老公的报告格式

```
📚 第{N1}-{N2}章学习完成

产出：
- 章节分析：10个文件 ✅
- 综合分析：1份 ✅
- 新Skill：{X}个（列出名称）
- 更新文件：REGISTRY.md / writer-agent.md / skill-matcher.md

下一步：第{N3}-{N4}章
```
