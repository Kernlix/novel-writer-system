# 📚 学习部门 (Learning Department)

## 部门使命
持续提升整个系统的创作能力。通过外部学习和内部分析形成知识→系统升级的闭环。

## 下属Agent
| Agent | 职责 |
|:------|:------|
| external-study-agent | 外部学习：优秀作品多Agent并行分析、技法提取 |
| internal-analysis-agent | 内部分析：用户反馈、创作结果、审核报告→改进建议 |

## 外部学习强制协作流程

> ⚠️ 强制规则：本流程不可跳过、不可简化。学习完成后没有系统升级 = 流程未完成。
> ⚠️ 当有新的学习Agent加入时，本流程必须同步更新，将其纳入协作。

```
🏢 负责人把握大方向
  ├── 用户提供学习材料
  └── 确认学习目标 + 派发任务

📚 学习部门启动（6个Agent并行分析）
  ├── ✍️ 写手Agent → 场景/对话/叙事技法
  ├── 📖 剧情Agent → 情节/节奏/伏笔
  ├── 👤 角色Agent → 人物/弧光/关系
  ├── 🏗️ 世界观Agent → 设定/力量体系
  ├── 🎨 润色Agent → 文风/语言/去AI化
  └── 🔬 质检Agent → 设定逻辑/一致性

  每个Agent完成三轮递进：
    ① 分析 → ② 提炼 → ③ 自我提升 → ④ 系统升级（填写upgrade字段）

  （各Agent产出直接进入审查）

🔍 审查官审查学习成果
  ├── 技法之间是否重叠？
  ├── 是否与已有知识重复？
  ├── 每个技法是否带有upgrade字段？
  ├── ✅ 全部通过 → 进入下一步
  └── ❌ 缺upgrade字段 → 打回补充

🏢 用户确认 → 写入知识图书馆
  ├── 通用技法 → knowledge/<类别>/learned/
  └── Agent专用 → knowledge/<agent>/learned/

⚠️ 强制系统升级（必做，不可跳过）
  ├── 根据upgrade字段更新对应agents/skills/hooks文件
  └── 记录到 .story-system/upgrade-log.md

🏢 负责人汇总报告 → 提交用户
  ├── 📚 学习了什么作品
  ├── 📝 提取了N个技法
  ├── ⚙️ 升级了M个系统文件
  └── 📎 升级日志已记录
```

## 内部分析流程

```
🏢 负责人接收指令 → 确认分析范围
  ↓
📊 内部分析Agent读取数据
  ├── 用户反馈记录
  ├── 近期审查报告
  └── 创作进度数据
  ↓
📊 输出改进建议报告 → 负责人审阅
  ├── ✅ 采纳 → 派发对应部门执行
  └── ❌ 不采纳 → 说明原因
  ↓
🏢 负责人汇总报告 → 提交用户
```

## 进化闭环

```
创作 → 审核 → 学习 → 招募 → 升级 → 创作（下次更强大）
```

每次学习完成后，系统获得新能力，下一次创作时就能使用。**没有升级日志 = 学习未完成。**
