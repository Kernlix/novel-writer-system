# 三层不同步

## 定义
改审查维度时不改prompt模板和gates字段，文档/代码/执行三层各版本

## 典型场景
- 审查12项 vs 模板2不匹配
- gates字段名不一致
- policy vs 实际不一致

## 自动检查
```bash
grep -c '维度' reviewer-agent.md # 审查维度数
grep -c '^[0-9]' lcm-rag-prompt-templates.md # 模板2检查项数
# 应一致
```

## 历史同类型错误
- 2026-07-11_prompt-upgrade

## 对策
1. 发生该类错误时，立即记录到 `knowledge/errors/entries/`
2. 更新 `root-causes.json` 计数
3. 更新 `README.md` 速查表
4. 将自动检查脚本写入 `pre-commit-check.sh`

## 系统教训
- 该类错误已发生 2 次，证明光靠记忆不可靠
- 必须通过自动化脚本+pre-commit检查来阻断
