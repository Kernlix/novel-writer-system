# 章节修改流程

> ⚠️ 任何修改（标题/正文/角色档案等）都必须走以下流程。

```
第1步：负责人确定修改内容
  ├── 明确修改范围
  ├── 读取当前版本文件
  └── 构建修改方案

第2步：→ 派发给 写手智能体 执行修改
  ├── 传入：当前版本 + 修改方案
  ├── 写手执行修改
  └── 输出：修改后版本

第3步：→ 派发给 审查官审查 ⚠️（打回循环）
  ├── 传入：修改后版本
  ├── 审查维度：修改是否正确？是否引起新矛盾？是否还有遗漏？
  │
  ├── ✅ 通过 → 报告负责人，进入第4步
  │
  └── ❌ 有问题 → 打回第2步，循环直到全部通过

第4步：负责人汇总报告
  ├── 📄 修改内容说明
  ├── 🔄 修改轮次：共X轮 / 打回Y次 / 最终全部通过
  ├── 📊 状态更新
  └── 📨 提交用户审阅
```
