# 🧩 子智能体隔离机制

## 概述
每个子智能体在独立上下文中运行，不互相干扰。

## 隔离层级

```
主智能体（全局上下文）
 ├── 子智能体 A（任务：大纲规划）
 │    └── 上下文：仅加载大纲相关文件
 ├── 子智能体 B（任务：章节写作）
 │    └── 上下文：仅加载当前章+角色卡片
 └── 子智能体 C（任务：质量审查）
      └── 上下文：仅加载审查规则+章节正文
```

## 实现方式
每个子智能体调用时：
1. 创建临时工作目录：`.workbench/<task-id>/`
2. 只复制任务所需文件到该目录
3. 子智能体运行完毕后返回结果
4. 自动清理临时目录

## 隔离规则

| 资源 | 主智能体 | 子智能体 A | 子智能体 B |
|:--|:--:|:--:|:--:|
| 项目全局设定 | ✅ 可读写 | ❌ 只读 | ❌ 只读 |
| 当前章节文件 | ✅ 可读写 | ❌ 不可见 | ✅ 可读写 |
| 角色档案 | ✅ 可读写 | ❌ 不可见 | ✅ 只读 |
| 审查报告 | ✅ 可读写 | ❌ 不可见 | ❌ 不可见 |
| RAG 数据库 | ✅ 可读写 | ✅ 只查询 | ✅ 只查询 |

## 配置文件
```yaml
# subagent-isolation.yaml
tasks:
  outline:
    read: [大纲/, 设定集/]
    deny: [章节/, 审查报告/]
  write:
    read: [大纲/当前卷.md, 人物/]
    write: [章节/]
    deny: [审查报告/]
  review:
    read: [章节/, 审查规则/]
    write: [审查报告/]
    deny: [人物/]
```
