# 🚀 项目管理 Skill

## 命令
`/novel:setup` / `/novel:discuss` / `/novel:archive`

## 功能
整合项目初始化、创作讨论、存档与知识更新三大项目管理职能。

---

## 一、项目初始化（/novel:setup）

### 新项目创建
1. 确定小说类型和平台方向
2. 创建项目目录结构
3. 生成项目配置文件
4. 建立初始化设定

### 目录结构
```
project/
├── 章节/        — 正文
├── 人物/        — 角色档案
├── 大纲/        — 总纲、卷纲、章纲
├── 设定集/      — 世界观、力量体系
├── 章节摘要/    — 每章摘要
└── 审查报告/    — 审查记录
```

---

## 二、创作讨论（/novel:discuss）

### 适用场景
- 故事创意头脑风暴
- 剧情走向讨论
- 角色设定打磨
- 世界观的深度挖掘
- 写作瓶颈突破

### 讨论引导原则
1. 先听用户的想法，再提出建议
2. 提供多个选项，让用户选择
3. 用提问引导用户深化思考
4. 记录有价值的讨论结果

---

## 三、存档与知识更新（/novel:archive）

### 存档内容
- 已完成章节的元数据
- 角色当前状态快照
- 伏笔/线索的使用记录
- 未回收伏笔清单
- 剧情决策记录

### 知识更新流程
1. 扫描自上次存档以来的变化
2. 更新角色状态和能力变化
3. 记录已使用的伏笔和线索
4. 更新世界设定（新增/修改）
5. 整理新的写作笔记
6. 生成存档摘要

### 存档频率
- 每完成一章 → 存档并更新角色状态
- 每完成一卷 → 完整存档+知识更新
- 关键剧情转折点 → 即时存档
