# 长篇小说RAG配置指南

> 基于用户输入的「长篇小说写作最优方案：LCM长上下文模型+轻量化RAG」整理。

---

## 一、RAG分块规则（小说专用，大粒度）

| 文档类型 | 分块策略 | 目标块大小 | 说明 |
|:--------|:---------|:----------|:-----|
| **人物卡** | 一张人物卡=一个块 | 3k~8k 字符 | 保持角色档案完整，不切碎人物背景/性格/关系 |
| **章节正文** | 一章=一个块 | 3k~8k 字符 | 保持叙事连贯，保证伏笔/对话不断裂 |
| **世界观设定** | 单一大规则=一个块 | 3k~6k 字符 | 如修炼体系、国家历史、种族规则各独立成块 |
| **分卷大纲** | 一卷大纲=一个块 | 5k~10k 字符 | 保持整卷剧情脉络完整 |
| **短文档**（< 2000字符） | 不拆分 | — | 整份作为一块 |

**重叠值：** 500 字符（约 300 token），保证相邻块之间的伏笔不因切块断裂。

> 💡 引擎 `engine.py` 已实现类型感知分块：`characters` 默认 5000，`chapters`/`outlines`/`settings` 默认 3000。

---

## 二、检索参数

| 参数 | 推荐值 | 说明 |
|:----|:------|:-----|
| **Top-k** | **3~4** | 不要一次拉十几份设定，过多设定会干扰模型剧情发挥 |
| **Reranker Top-n** | 5~8 | Reranker过滤后保留的候选数，可略高于Top-k供模型选择 |
| **双层模式粗召回** | 10 | 方案B第一层粗检索量，第二层筛选后保留3~5份 |

---

## 三、不同写作场景搭配建议

| 场景 | 推荐方案 | 说明 |
|:----|:--------|:-----|
| **网文日更、单主线长篇** | 模式A：128K LCM + 本地RAG | 成本最低，续写稳定，每天快速出稿 |
| **实体出版长篇、严谨权谋/历史** | 模式B：双层递进RAG + 超长LCM | 定期用LCM读整卷自查漏洞，统一人物行为逻辑 |
| **短篇、十万字以内小长篇** | 简化：单用128K LCM | 临时粘贴设定即可，无需搭建RAG；超20万字再补RAG |
| **多开小说、同时写两本长篇** | RAG分两个独立向量库 | ChromaDB不同collection前缀，切换小说时切换知识库 |
| **群像/多势力/多时空史诗** | 模式B + 双层过滤 | 粗召回Top10 → 过滤3~5份 → 送超长LCM |

## 四、本地部署推荐组合（写小说性价比最高）

| 组件 | 推荐 | 说明 |
|:----|:-----|:-----|
| 长上下文模型 | **Qwen2-7B/14B-128K** | 本地显卡可跑，6G显存起步 |
| 嵌入模型 | **qwen3-embedding:4b** | 已配置于 `engine.py`，Ollama运行 |
| 向量库 | **ChromaDB** | 已集成，无额外服务，开箱即用 |
| Reranker | **Qwen3-Reranker-0.6B** | 已实现独立服务（`server_reranker.py`） |
| 框架 | 轻量自实现 + ChromaDB API | 无需LlamaIndex/LangChain等重框架 |

### 模型升级路线

```
当前：qwen3-embedding:4b (Ollama)
  → 未来可升级到：bge-m3 / bge-large-zh-v1.5 (检索精度更高)

当前：Qwen3-Reranker-0.6B (独立服务)
  → 未来可升级到：bge-reranker-v2-m3 (精度提升)
```

## 五、写作高频实用功能速查

| 功能 | 怎么做 |
|:----|:-------|
| **剧情自查** | RAG查设定+章节原文 → 模板2复盘审查 → LCM批量排查bug |
| **支线梳理** | RAG召回所有配角档案+对应章节 → LCM整理完整支线时间线 |
| **统一文风** | 导入前10章全文存入RAG（整章一块不拆分），续写时自动匹配 |
| **新增人物** | 新建人物文档 → 重建RAG索引 → 后续写作自动读取，无需重复粘贴设定 |
| **Anti-hallucination（幻觉防止）** | 参考资料无记载的信息一律不写，如有缺失直接标注，禁止自行补充设定 |

| 用户方案 | 灵境实现 |
