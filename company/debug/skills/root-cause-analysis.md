# 根因分析法 Skill

## 7类根因定位问题链

### 1. 改了A不改B
**追问：** 这个改动影响了哪些文件？所有引用旧值的文件都更新了吗？
**检查：** `grep -rn "旧值" --include="*.md" .`

### 2. 声明没实现
**追问：** 说新增的文件存在吗？hot的流程能走通吗？
**检查：** `[ -f "声明的文件" ]`

### 3. 脚本静默失败
**追问：** 批量操作有计数吗？改前改后数值一致吗？
**检查：** BEFORE=$(grep -c "模式" 文件); AFTER=$(grep -c "模式" 文件)

### 4. 路径泄露
**追问：** 代码/文档里有绝对路径或用户名吗？
**检查：** `grep -rn "D:\\\\allproject" --include="*.md" .`

### 5. 创建不验证
**追问：** 新文件建完有跑过吗？
**检查：** 脚本必须实际执行一次

### 6. 三层不同步
**追问：** 改审查维度时同步改了prompt模板和gates吗？
**检查：** 三者维度数应一致

### 7. 子智能体不闭环
**追问：** Agent交付了三问吗？（文件→注册→计数）

## 使用
发现错误 → 逐类追问 → 归类 → 按对应categories/模板修复
