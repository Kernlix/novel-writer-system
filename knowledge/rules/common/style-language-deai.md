# 文风语言+去AI化 分析维度参考清单

> 本清单供学习部门在分析作品时参考，用于系统评估作品的文风语言水平和去AI化程度。
> 适用于 `/novel:learn` 流程中 writer 和 polish Agent 的文风语言维度分析。

---

## 一、句式多样性分析

### 核心问题

长短句搭配是否自然？是否存在「完美对称句式」等AI写作特征？句式的变化是否服务于节奏与情绪？

### 检查要点

| # | 检查项 | 错误表现（AI味） | 正确做法（人类感） |
|:-:|:------|:----------------|:-------------------|
| ① | **长短句交替** | 所有句子长度接近（如持续15-25字/句），节奏单调 | 长句（30+字）展开细节→短句（5-10字）制造冲击→中句过渡，形成呼吸感 |
| ② | **完美对称句式** | 大量「不仅…而且…」「既…又…」「虽然…但是…」等对仗结构 | 拆散对称：「他没生气，还笑了」代替「他不仅没有生气，反而笑了」 |
| ③ | **排比适度** | 连续3句以上同结构排比（AI常见套路） | 排比不超过2句，或在第三句打破结构 |
| ④ | **段首多样性** | 每段都以「然而/不过/与此同时/另一方面」开头 | 段首用内容自然衔接，或用短词/角色动作/环境描写开头 |
| ⑤ | **句式结构多样性** | 所有句子都是「主语+谓语+宾语」的常规结构 | 混合使用：倒装句、插入语、破折号、省略句、独词句 |
| ⑥ | **标点使用习惯** | 逗号句号循环，缺乏破折号/分号/省略号等调节节奏的标点 | 破折号用于打断/转折，省略号用于欲言又止，分号用于并列节奏 |

### 句式多样性评分方法

1. 随机取连续300字段落
2. 统计：最短句字数、最长句字数、平均句长
3. 计算「句长标准差」——标准差<5说明句式过于均匀（AI特征）
4. 统计对称句式出现频率：每1000字>3处即报警

## 二、角色对话差异化分析

### 核心问题

每个角色的说话方式是否有明显区别？盖住角色名能否猜出是谁在说话？

### 检查要点

| # | 检查项 | 错误表现 | 正确做法 |
|:-:|:------|:---------|:---------|
| ① | **句长特征** | 所有角色平均句长接近（差异<15%） | 每个角色有专属句长特征：急性子短句（4-8字）、沉稳者中句（10-18字）、话痨长句（20+字） |
| ② | **语气词指纹** | 角色不用语气词或滥用同一套语气词 | 分配专属语气词：A爱用「嘛/啦/哟」，B总说「嗯/啊」，C的口头禅是「那个…」 |
| ③ | **惯用语/口头禅** | 角色无标志性重复表达 | 核心角色设计1-2个重复梗/口头禅（如：「这不科学」「你行你上」「……大概」） |
| ④ | **语体差异** | 所有角色使用同一语体（都正式或都随意） | 分层设计：高身份角色用敬语/正式语，平民角色用口语/俚语，学者角色用书面语 |
| ⑤ | **情绪表达方式** | 所有角色用同样方式表达愤怒/开心/悲伤 | 不同角色有不同情绪表达模式：A生气时沉默，B生气时嘲讽，C生气时摔东西 |
| ⑥ | **对话打断/省略** | 对话轮次之间无打断、无省略、无抢话 | 自然对话应有：打断（「等等——」）、省略（「那个…」）、抢话（「不是，我是说——」） |

### 角色语音签名设计参考

| 角色类型 | 句长 | 语气词 | 惯用语 | 情绪特征 |
|:---------|:----:|:-------|:-------|:---------|
| 急性子/易怒 | 短（3-8字） | 少用语气词，多用叹词 | 「搞什么！」「喂！」 | 愤怒→喊/拍桌；悲伤→砸东西 |
| 温和/沉稳 | 中（10-16字） | 「嗯」「好」「……吧」 | 「让我想想」「不着急」 | 愤怒→沉默/冷处理；悲伤→沉默 |
| 话痨/兴奋 | 长（18-30字） | 「啊」「呢」「嘛」「哟」 | 「话说回来」「你猜怎么着」 | 愤怒→更多话/讽刺；悲伤→不停说话掩饰 |
| 高冷/寡言 | 短（2-6字） | 几乎不用 | 沉默是主要回应 | 愤怒→冷笑一声；悲伤→消失/独自待着 |
| 天真/活泼 | 中短（5-12字） | 「耶」「哇」「啦」「嘛」 | 「好厉害！」「真的假的！」 | 喜怒哀乐都写在脸上，用语气词调节 |

## 三、去AI化写作分析

### 核心问题

这篇作品的语言为什么感觉「像人写的」而不是AI写的？有哪些AI写作痕迹需要检测？

### AI高频词检测

以下词汇在AI生成文本中出现频率远高于人类写作。检测标准：出现即标记，每处记1分。

#### 🔴 P0级：出现即报警（AI特征最明显）

| 词汇 | 替换建议 |
|:-----|:---------|
| 凸显/彰显 | 显得/一看就/明摆着 |
| 无疑/毋庸置疑 | 肯定/不用说/谁都知道 |
| 换言之/换句话说 | 删掉或用「也就是说」 |
| 本质上/归根结底 | 说白了/其实 |
| 不禁/不由得 | 删掉或用「下意识」 |
| 愈发/愈发显得 | 更/越来越 |
| 暗自 | 心里/偷偷 |
| 某种/某种程度 | 删掉或换成具体描述 |

#### 🟡 P1级：高频率需警惕

| 词汇/结构 | 问题说明 |
|:----------|:---------|
| 微微（皱眉/点头/摇头） | 万能修饰词，每千字>3次即报警 |
| 轻轻（说/拍/放） | 同上，建议替换为具体动作 |
| 似乎/仿佛/好像 | 不确定性词，过量使用削弱叙述力度 |
| 有些/略显/稍显 | 模糊修饰词，建议删或替换为具体描写 |
| 然而/但是/因此/不过/于是 | 过度连接词，每500字>5个即报警（参考 anti-ai-polish.md） |

### 完美对称句式检测

对称句式（对仗结构）是AI写作的核心特征之一。检测方法：

```
匹配模式：
  - 「不仅…而且…」 / 「不仅没有…反而…」
  - 「既…又…」 / 「既不会…也不会…」
  - 「虽然…但是…」 / 「虽然…却…」
  - 「越是…就越…」
  - 「无论…都…」
  - 「一方面…另一方面…」

报警阈值：每1000字 > 3处
改写方法：拆散对称 → 保留一半语义，另一半用破折号/逗号/句号断开
```

### 其他AI痕迹检测

| # | 检测维度 | 检测方式 | 报警条件 | 参考 |
|:-:|:---------|:---------|:---------|:-----|
| 1 | **通用描写** | 检测缺乏具体细节的概括性描述（「他看起来很生气」） | 每段 ≥ 1处 | anti-ai-polish.md |
| 2 | **直白情感** | 检测直接说出情感词（「他很伤心」「她感到幸福」） | 每500字 > 3处 | anti-ai-polish.md |
| 3 | **对话过完整** | 单次发言 > 30字的比例；对话轮次无打断/省略 | > 40% 或连续3轮无打断 | anti-ai-polish.md |
| 4 | **叙述客观化** | 段落缺乏叙述者主观情绪/吐槽 | 每300字无叙述者态度 | anti-ai-polish.md |
| 5 | **角色同质化** | 不同角色对话的平均句长、语气词使用率差异 < 15% | 差异过小 | anti-ai-polish.md |
| 6 | **心口一致** | 角色言行无反差（无内心独白vs台词的拆台效果） | 核心角色连续2次无反差 | anti-ai-polish.md |
| 7 | **逻辑衔接过度** | 段首过度使用「然而/不过/与此同时/另一方面」 | 连续3段段首均有 | anti-ai-polish.md |

### 去AI化写作评分方法

| 维度 | 弱（1分） | 中（3分） | 强（5分） |
|:-----|:---------|:---------|:---------|
| 句式自然度 | 大量对称句式、排比、AI高频词 | 偶有AI痕迹但整体自然 | 几乎无AI痕迹，读起来像自然人书写 |
| 对话真实度 | 角色台词无区别，轮次完整无打断 | 有基本区分，有打断/省略 | 每角色有语音签名，对话真实如日常交谈 |
| 情感表达 | 直接说出情感词 | 部分用行动/细节代替 | 完全用行动/细节/潜台词传递情感 |
| 叙事温度 | 客观中立如新闻报道 | 有叙述者态度但偏弱 | 有强烈叙述者声音（吐槽/自嘲/刻薄/温柔） |

## 四、叙事节奏分析

### 核心问题

描写/对话/行动的比例是否合理？节奏是否符合场景情绪需求？

### 检查要点

| # | 检查项 | 错误表现 | 正确做法 |
|:-:|:------|:---------|:---------|
| ① | **场景-节奏匹配** | 战斗场景大段描写环境，对话场景全程无动作穿插 | 战斗→短句+快节奏+行动为主；对话→中速，穿插动作/内心；抒情→慢节奏，描写为主 |
| ② | **描写/对话/行动比例** | 比例失衡（如70%描写30%对话，或60%对话0%描写） | 标准场景配比：对话40-50%，行动20-30%，描写20-30%（抒情类可调高描写比例） |
| ③ | **节奏高低起伏** | 全章同一节奏，无快慢变化 | 紧张场景后接缓和场景，信息量大后接轻快对话，形成呼吸感 |
| ④ | **段落长度控制** | 所有段落长度一致 | 紧张→短段落（1-3句），舒缓→中段落（3-6句），信息密集→长段落（6-10句） |
| ⑤ | **字数分配** | 高光时刻字数不足，铺垫过度 | 高潮场景（30-40%）、发展场景（25-35%）、铺垫场景（15-20%）、收尾场景（10-15%） |

## 五、综合分析方法

### 四步分析法

```
第1步：抽取300字段落 → 做句式多样性检测（句长标准差、对称句式频率）
第2步：统计角色对话 → 计算每位角色的平均句长/语气词频率/惯用语 → 差异化是否明显？
第3步：运行AI痕迹扫描 → 检查P0/P1级AI特征词、对称句式、过度连接词 → 标记位置
第4步：分析叙事节奏 → 统计描写/对话/行动比例 → 判断节奏与场景情绪是否匹配
```

### 本能化建议

分析完成后，将发现的问题/优秀技法提炼为 style domain 的本能：

| 发现 | 本能示例 |
|:-----|:---------|
| 句式多样但偶尔对称句式 | `style:break-parallel-structure` — 打破对称句式 |
| 某角色对话特别有辨识度 | `style:dialogue-signature-<角色>` — 角色语音签名 |
| AI高频词出现 | `style:avoid-ai-buzzwords` — 避免AI高频词 |
| 节奏控制优秀 | `style:pacing-ratio-<场景类型>` — 节奏配比参考 |
