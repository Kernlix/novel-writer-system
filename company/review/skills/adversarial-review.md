# ⚔️ 对抗审查 Skill

## 命令
`/novel:review:adversarial`

## 功能
用第二个审查Agent独立审读已通过的章节，通过视角切换和维度侧重差异化，发现主审查遗漏的问题。

## 核心原则

### 1. 独立性
- 两个Agent**不共享中间状态**
- 各自独立读取章节正文和规则库
- 各自独立输出审查报告

### 2. 视角切换
- 主审查按标准顺序：叙事 → 角色 → 情节 → 语言 → 一致性
- 对抗审查按逆序/随机顺序：一致性 → 语言 → 情节 → 角色 → 叙事
- 维度权重不同，确保盲区不重叠

### 3. 倾向性引导
| 维度 | 主审查倾向 | 对抗审查倾向 |
|:-----|:-----------|:-------------|
| 设定一致性 | 宽松 | 严格 |
| 角色行为 | 整体合理 | 极端情况 |
| 伏笔 | 已回收 | 遗漏未收 |
| 情感 | 脉络清晰 | 断裂点 |
| 节奏 | 整体健康 | 局部疲劳 |

## 比对流程

```
主审查通过
  ↓
对抗审查独立执行
  ↓
两报告比对
  ├── 交集 → 高可信度问题
  ├── 对抗独有 → 中可信度，标记「对抗发现」
  └── 结论相反 → 人工仲裁
  ↓
合并输出 → 最终审查结论
```

## 输出格式

```json
{
  "chapter": "第XX章",
  "origin": "adversarial",
  "gates": { /* 同主审查12项 */ },
  "new_issues": [
    {"confidence": "high/medium/low", "item": "主审查遗漏的问题", "fix": "建议"}
  ],
  "conflicts": [
    {"dimension": "冲突维度", "reviewer": "主审查结论", "adversarial": "对抗结论"}
  ]
}
```

## 适用场景
- 长篇连载的关键转折章（第5、10、15…章强制触发）
- 第一次审查存在任何 `medium` 或低确信度标记的问题
- 跨卷衔接章节，需特别确保一致性
- 涉及复杂世界观/多条伏笔线的章节

## 注意事项
- 不增加打回率，只提高漏检捕获率
- 对抗审查耗时控制在主审查的 60% 以内
- 对抗发现的 `low` 级问题自动归入「建议优化」而非「必须修改」
