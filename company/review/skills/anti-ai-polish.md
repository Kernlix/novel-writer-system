# 🎨 去AI味完整流水线 Skill

## 概述

基于 `polish-agent.md` 的13项检测维度，升级为完整的**四步流水线**，可独立运行或嵌入 post-write / post-review hook 自动触发。

## 流程总览

```
输入正文
   │
   ▼
① 检测 ── 扫描常见AI套话/废话模式，输出问题清单
   │
   ▼
② 标记 ── 标出需要改写的句子/段落（含原因说明）
   │
   ▼
③ 改写 ── 自动替换为自然人类表达
   │
   ▼
④ 验证 ── 对比改写前后的差异，确认去AI化效果
   │
   ▼
输出改写后正文 + 差异报告
```

---

## ① 检测（Scan）

### 检测维度

| # | 维度 | 检测方式 | 报警条件 |
|:--|:-----|:---------|:---------|
| 1 | **过度连接词** | 统计"然而/但是/因此/不过/虽然/尽管/于是/从而"密度 | 每500字 > 5个 |
| 2 | **对称句式** | 检测"不仅…而且…""既…又…""虽然…但是…"等对称结构 | 每1000字 > 3处 |
| 3 | **完美排比** | 连续3句以上同结构的排比句 | 出现即报警 |
| 4 | **通用描写** | 检测缺乏具体细节的概括性描述（如"他看起来很生气"） | 每段含 ≥ 1处 |
| 5 | **直白情感** | 检测直接说出的情感词（"他很伤心""她感到幸福"） | 每500字 > 3处 |
| 6 | **对话过完整** | 单次发言 > 30字的比例；对话轮次无打断/省略/抢话 | > 40% 或 连续3轮无打断 |
| 7 | **叙述客观化** | 段落缺乏叙述者主观情绪/吐槽 | 每300字无叙述者态度 |
| 8 | **角色同质化** | 不同角色对话的平均句长、语气词使用率差异 < 15% | 差异过小 |
| 9 | **心口一致** | 角色言行无反差（无内心独白vs台词的拆台效果） | 核心角色连续2次无反差 |
| 10 | **缺乏标志性重复** | 角色无标志性口头禅/惯用语 | 出场 > 3次的角色无签名 |
| 11 | **AI高频词** | 检测"凸显/彰显/无疑/换言之/本质上/愈发/愈发显得/不禁/暗自/不由得" | 出现即标记 |
| 12 | **万能修饰词** | 检测"微微/轻轻/似乎/仿佛/有些/某种/略显" | 每1000字 > 8个 |
| 13 | **逻辑衔接过度** | 段首过度使用"然而/不过/与此同时/另一方面"等过渡词 | 连续3段段首均有 |

### 输出格式
```
[问题清单]
- 第15行 | 过度连接词 | "然而" x3 | 建议：删除或替换为"可"
- 第42行 | 完美排比 | "他…他…他…" | 建议：打破排比结构
...
```

## ② 标记（Mark）

对检测出的每个问题，在原文中**用注释括号标注**：

```
【去AI: 过度连接词】然而，他并没有因此放弃。
→ 标记为待改写

【去AI: 对话过完整】"我觉得这件事情可能还有另一种可能性，毕竟我们之前都没有考虑过这个角度。"
→ 标记为待缩短/打断

【去AI: 直白情感】她感到非常伤心。
→ 标记为改为行动/细节展示
```

同时对改写**优先级**分级：
- 🔴 **P0-必须改**：AI高频词、完美排比、对话过度完整、叙述客观化
- 🟡 **P1-建议改**：过度连接词、对称句式、直白情感、角色同质化
- 🟢 **P2-可选改**：通用描写、缺乏标志性重复、万能修饰词

## ③ 改写（Rewrite）

### 改写策略对照表

| 问题类型 | 改写策略 | 示例 |
|:---------|:---------|:------|
| 过度连接词 | 删掉或替换为口语化表达 | "然而" → "可"/"但" / 直接删除 |
| 对称句式 | 拆散或保留一半 | "不仅没有生气，反而笑了" → "他没生气，还笑了" |
| 完美排比 | 打破结构，保留语义 | "他跑步，他游泳，他骑行" → "他跑步游泳骑行，三项轮着来" |
| 通用描写 | 增加具体感官细节（视/听/触/嗅/味） | "天气很热" → "柏油路面蒸起热浪，蝉鸣像电钻" |
| 直白情感 | 改为行动/细节/比喻 | "她很伤心" → "她盯着窗外发呆，手指无意识地抠着桌角" |
| 对话过完整 | 缩短为 ≤2句/轮，插入打断/抢话/省略 | 详见"短句攻防节奏" |
| 叙述客观化 | 嵌入叙述者主观态度 | "他走进房间" → "他走进房间——啧，这品味" |
| 角色同质化 | 分配专属句式/语气词/句长特征 | 详见"角色语音签名设计" |
| 心口一致 | 插入内心独白拆台 | "没问题！"（鬼才信没问题） |
| 缺乏标志性重复 | 设计2-3个重复梗/口头禅 | 角色A:"这不科学…" 角色B:"你行你上" |
| AI高频词 | 直接替换为日常用语 | "凸显" → "显得"/"一看就" |
| 万能修饰词 | 删除60%，保留的替换为具体描写 | "微微皱眉" → "眉头拧成一团" |
| 逻辑衔接过度 | 删掉段首过渡词，用内容自然衔接 | "然而，就在此时" → "就在这时" |

### 素晴学专项改写

参考 `polish-agent.md` 润色策略：
1. **短句攻防节奏** — 对话每轮 ≤2句，连续3轮无打断则插入截断式回应
2. **吐槽式叙述** — 每300字嵌入叙述者情绪评价（刻薄/自嘲/阴阳怪气）
3. **角色语音签名** — 词法/句法/语用三层指纹
4. **内心独白vs台词反差** — 用破折号或括号插入拆台式内心吐槽
5. **重复梗系统** — 核心角色2-3个标志性重复锚点

## ④ 验证（Verify）

### 验证方式

**A. 对比报告**：改写前后差异对比（行级diff）

```
[改写差异报告]
+ 第15行: "可他没有放弃"  ← 原为 "然而，他并没有因此放弃"
+ 第42行: "他跑步游泳骑行，三项轮着来"  ← 原为 "他跑步，他游泳，他骑行"
+ 第78行: "眉头拧成一团"  ← 原为 "微微皱眉"
[统计]
  原文总字数: 3580
  改写后字数: 3412
  改写处: 12处
  去AI得分: 82/100 (P0问题全部解决，P1剩余2处)
```

**B. 回检**：重新运行①检测，确认：
- P0问题：100% 解决
- P1问题：≥ 80% 解决
- 无新增AI痕迹（如过度使用口语反而产生新的套路化表达）

**C. 风险提示**：
- 过度改写可能丢失原文信息量 → 保留关键事实性描述
- 口语化过度可能破坏严肃场景氛围 → 根据场景调性调整改写力度
- 吐槽式叙述不适合 POV 限制叙事 → 仅限叙述者视角

## 执行方式

### 独立调用
在审查/润色环节直接执行本流水线：
```
运行 anti-ai-polish skill → 输入正文 → 输出改写后正文 + 差异报告
```

### Hook 集成
在 `post-write` hook 中加入质检项（见 `company/writing/hooks/post-write.md`），写完后自动触发检测，P0问题强制返回修改，P1/P2记录到报告中供作者参考。

### 与 polish-agent 的关系
- `polish-agent.md`：定义检测维度和润色策略（理论/策略层）
- `anti-ai-polish.md`：可执行的流水线流程（执行/操作层）
- 两者配合使用：策略来自 polish-agent，执行按本流水线
