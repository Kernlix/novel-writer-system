## 🎲 死亡游戏机制（death-game）

### srzd:layered-rule-design
```
id: "srzd:layered-rule-design"
trigger: "设计死亡游戏/生存游戏/博弈类小说的规则体系时"
action: "用三层规则设计：表面规则（让读者和角色先理解）→ 隐藏规则（通过情节发展逐步揭示）→ 终极规则（最终反转，颠覆前两层）。每一层规则的揭示都是一个叙事高潮。"
confidence: 0.9
domain: "death-game"
scope: "global"
source: "《十日终焉》第2章——表面规则"找说谎者"，隐藏规则"身份牌强制说谎"，终极规则"所有人都是说谎者""
```

**本能说明：** 死亡游戏的核心吸引力在于规则的层次感。不一次性公布所有规则，而是通过角色行动逐步揭示——第一层规则驱动前半段情节，隐藏规则在中段制造转折，终极规则在高潮处颠覆一切。每层规则本身要有内在逻辑自洽性。

### srzd:multi-layer-reversal
```
id: "srzd:multi-layer-reversal"

action: "用三层反转递进：第一层（表面问题）→ 第二层（为什么）→ 第三层（根本性颠覆）。每一层反转都比上一层更深刻。"
confidence: 0.9
domain: "suspense"

```

**本能说明：** 多层反转的力量在于递进：第一层反转回答"是什么"，第二层回答"为什么"，第三层回答"我们是什么"。每一层都比上一层更深刻，最终的反转不是"信息"而是"存在"的颠覆。这创造了极强的叙事深度。

confidence: 0.85
domain: "suspense"
scope: "global"
source: "《十日终焉》第6章——韩一墨的极简讲述"什么都不知道"，无法验证也无法推翻"
