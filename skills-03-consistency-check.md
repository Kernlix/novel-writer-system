# 🔗 一致性检查 Skill

## 命令
`/novel:check`

## 功能
跨章节/跨卷的一致性检查，确保长篇创作不出现矛盾。

## 检查维度

### 1. 时间线一致性
- 事件先后顺序
- 角色年龄/时间流逝
- 季节/天象与描述一致

### 2. 世界观一致性
- 力量体系规则
- 地理描述（方向/距离/位置）
- 历史事件

### 3. 角色一致性
- 性格是否稳定（或成长轨迹合理）
- 能力变化有依据
- 关系变化有铺垫

### 4. 伏笔一致性
- 已埋设伏笔是否回收
- 未回收伏笔是否仍有效
- 新信息是否与旧设定矛盾

### 5. 角色成长轨迹检查
- 角色的变化是否有充�的铺垫？
- 每个成长阶段是否跨越足够的章节（3章+）？
- 成长路径上是否有反复/退步（而非一路向上）？
- 外部事件是否推动了内部变化？

### 6. 节奏健康度检查
- [ ] 最近几章是否有情感起伏？
- [ ] 是否连续3章以上无任何冲突/进展？
- [ ] 幽默与严肃的比例是否平衡？
- [ ] 视角切换是否自然，是否让读者困惑？

### 7. 时代一致性检查
需加载 `.era-knowledge/` 知识库对照审查：
- **物质**：武器/装备/建筑/交通工具/日常用品是否超纲
- **制度**：管理/军事/商业模式是否跨时代
- **概念**：科学知识/数学应用是否越界
- **语言**：角色对话/叙述者语言是否使用了超越认知的词汇

输出格式：按 🔴 明显违和 / 🟡 需合理化 / 🟢 合理可用 分级标注。

### 8. 逻辑漏洞检查
- [ ] 情节发展是否符合内在逻辑？
- [ ] 角色行为动机是否合理？
- [ ] 能力/规则是否被滥用或前后矛盾？
- [ ] 时间线是否自洽？
- [ ] 是否存在信息不一致（角色知道了不该知道的事）？
