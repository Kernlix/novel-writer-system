# ⚡ 能力反转诅咒 (Ability Inversion Curse)

> "不死"是比"死亡"更可怕的惩罚——力量的反面往往是诅咒。

## 核心原理

角色使用同化/夺取类能力试图控制目标 → 目标拥有更高级的同类能力 → 角色被反向同化 → 主动能力变成被动诅咒。

### 效果
- 将"力量获取"的叙事翻转为"力量诅咒"
- 制造"以彼之道还施彼身"的戏剧冲击
- 角色的死亡/失败不是被打败，而是被自己的能力吞噬

## 技法结构

```
阶段1: 能力施展（角色主动使用同化能力）
  ↓
阶段2: 初步成功（目标出现伤口/变化——角色以为成功）
  ↓
阶段3: 异常发现（角色自己也出现了相同变化——困惑）
  ↓
阶段4: 反向同化确认（目标的"不灭"覆盖了角色的同化——绝望）
  ↓
阶段5: 诅咒锁定（角色无法松手/退出——被困在能力中）
  ↓
阶段6: 死亡/终结（角色以被同化的状态死亡）
```

## 使用指南

### 适用场景
- 能力对战中"以弱胜强"计划失败的转折
- 角色发现自己"偷鸡不成蚀把米"的悲剧时刻
- 力量体系中"克制关系"的实战展示
- 需要展示"能力天花板"的场景

### 注意事项
1. **反转必须有逻辑基础**：目标的同类能力必须比角色更强（等级差）
2. **阶段2的"虚假成功"至关重要**：让读者和角色一起以为计划成功
3. **诅咒的"锁定"要具体**：不是抽象的"被困住"，而是具体的物理约束（如"松手=死亡"）
4. **死亡方式要与诅咒一致**：角色以被同化的方式死亡，而非普通的物理死亡

### 变体
- **代价反噬型**：能力成功但代价超出预期（如获得力量但失去人性）
- **身份吞噬型**：能力成功但角色被目标的人格/记忆覆盖
- **双向诅咒型**：双方都被困在同化状态中，谁先松手谁死

## 来源章节分析

**第534章「同化」**：钱五握住玄武的手试图用"双生花"夺取身体。初步成功（双方出现相同伤口）→ 异常发现（伤口变黑无血）→ 反向同化确认（"不灭"覆盖"双生花"）→ 诅咒锁定（松手=死亡）→ 三人阵亡。

**关键设计**：玄武的"不灭"不是简单的"防御"，而是"反向同化"——她可以同化别人但别人无法同化她，构成极端的规则不对称。
