# 📖 拆文系统化 Skill — Story Deconstruction

> 隶属于学习部门。将优秀作品的阅读学习从「凭感觉看」升级为「系统化拆解」，
> 让每一次阅读都能沉淀为可复用的写作方法论。

## 命令

`/novel:deconstruct` — 启动系统化拆文学习

## 概述

拆文四步法：**扫榜 → 拆结构 → 拆人设 → 拆节奏**

```
输入 : 一部作品（全文或前N章）
流程: 扫榜(选材定位) → 拆结构(骨架) → 拆人设(血肉) → 拆节奏(呼吸)
输出: 结构化拆文报告 → 技法提炼 → 本能沉淀 → 系统升级
```

---

## 第一步：扫榜 (Scouting & Positioning)

**目标**：确定学习目标、理解作品在市场中的定位，建立全局认知。

### 操作指引

| 维度 | 分析内容 | 关键问题 |
|:-----|:---------|:---------|
| 平台定位 | 首发平台、分类标签、推荐机制 | 这套写法在这个平台上为什么能跑？ |
| 数据画像 | 总收藏/追读/月票/推荐票趋势 | 数据增长曲线有什么特征？什么节点爆发？ |
| 读者画像 | 读者评论关键词、打赏分布 | 读者最兴奋的点是什么？最不满的点是什么？ |
| 竞争对标 | 同期同类作品对比、差异化亮点 | 这本书在同赛道中靠什么胜出？ |
| 题材判断 | 题材热度周期、细分赛道成熟度 | 是跟风爆款还是开辟新赛道？ |

### 输出

```
## 扫榜报告

作品：《书名》(作者)
平台：起点/番茄/晋江/…
字数：XX万（连载/完结）
标签：[标签1, 标签2, ...]

### 市场定位
- 平台推荐机制适配：
- 数据特征：
- 读者兴奋点：
- 差异化优势：

### 学习目标
本次拆文重点学习：
1. 【结构】______
2. 【人设】______
3. 【节奏】______
```

## 第二步：拆结构 (Structural Deconstruction)

**目标**：解剖作品的骨架——卷/部划分、章节节奏、剧情结构、伏笔网络。

### 操作指引

| 维度 | 分析内容 | 关键问题 |
|:-----|:---------|:---------|
| 宏观结构 | 分卷/分部的节奏与转折点 | 每个大情节段的起承转合如何分布？ |
| 章节结构 | 每章的字数分布、开头技法、钩子密度 | 章节开头如何抓人？结尾如何留人？ |
| 情节类型 | 主线/支线/日常的比例分布 | 各类情节的穿插节奏是怎样的？ |
| 伏笔网络 | 伏笔的埋设-回收-再埋设 | 短中长伏笔的比例？回收节奏？ |
| 高潮布局 | 大高潮/中高潮/小高潮的间隔 | 爽点间隔多少章？如何层层递进？ |

### 分析方法

```
1. 画出全书情节脉络图（卷/部级别的起伏曲线）
2. 取样统计前30章的结构数据（每章字数、钩子、起承转合）
3. 标注伏笔节点 → 形成伏笔-回收映射表
4. 计算高潮间隔 → 总结爽点密度节奏
```

### 输出

```
## 结构拆解报告

### 宏观结构
```mermaid
%% 情节起伏曲线 (文字描述)
开篇(1-5章)   : 引入世界 → 主角登场 → 首次冲突
第一卷(6-20章) : 小团队建立 → 首次挑战 → 小高潮 → 新矛盾
第二卷(21-40章): 势力扩张 → 中期危机 → 大高潮 → 引入新角色
```

### 章节取样分析（前30章）
| 范围 | 平均字数 | 开头技法 | 钩子类型 | 结尾留人方式 |
|:-----|:---------|:---------|:---------|:-------------|
| 1-10章 | 2500 | 悬念首句 | 身份谜团 | 章末反转 |
| 11-20章 | 2800 | 环境切入 | 成长期待 | 戛然而止 |
| 21-30章 | 3000 | 对话开场 | 冲突爆发 | 悬念召唤 |

### 伏笔网络
| 伏笔 | 埋设章节 | 预期回收 | 状态 |
|:-----|:---------|:---------|:-----|
| 主角的身世秘密 | 第3章 | 第50章 | 未回收 |
| 反派的真实身份 | 第8章 | 第35章 | 已回收 |
| 宝物隐藏功能 | 第15章 | 第60章 | 未回收 |

### 爽点密度
- 大高潮间隔：每XX章
- 中高潮间隔：每XX章
- 小爽点频率：每章X个
- 密度曲线评价：[密集/平稳/稀疏]
```

## 第三步：拆人设 (Character Deconstruction)

**目标**：解剖角色的设计逻辑——从人设标签、成长弧线、关系网络三个层面。

### 操作指引

| 维度 | 分析内容 | 关键问题 |
|:-----|:---------|:---------|
| 人设标签 | 角色核心标签（3-5个），如「傲娇·天才·兄控」 | 标签如何组合产生化学反应？ |
| 出场设计 | 角色首秀方式、给人第一印象 | 首秀做了什么事让读者记住？ |
| 成长弧线 | 角色从A点→B点的转变轨迹 | 转折事件是什么？转变是否可信？ |
| 关系网络 | 角色之间的情感连接、权力关系、利益纽带 | 关系网如何推动剧情？ |
| 缺陷设计 | 核心缺陷、缺陷驱动的笑点/冲突/成长 | 缺陷在后续如何被利用？被克服？ |

### 输出

```
## 人设拆解报告

### 主角：[姓名]
| 维度 | 分析 |
|:-----|:------|
| 核心标签 | [X]·[Y]·[Z]（标签化合反应效果） |
| 出场设计 | 第X章首秀，通过[事件/对话/场景]立住人设 |
| 核心动机 | ______ |
| 成长弧线 | 初始→[事件A]→[事件B]→当前 |
| 核心缺陷 | ______（驱动哪些剧情和冲突） |
| 高光时刻 | 最圈粉的章节/事件 |
| 读者共鸣点 | |

### 配角：[姓名]
...

### 关系网络
- 主角↔配角A：[关系类型] → [对剧情的作用]
- 主角↔配角B：[关系类型] → [对剧情的作用]
- 配角的化学反应：[A和B之间的张力/配合]

### 人设技法提炼
1. **[技法名称]**：________________
   原文佐证：________________
   可复用方式：________________
```

## 第四步：拆节奏 (Pacing Deconstruction)

**目标**：解剖作品的气息与呼吸——叙事节奏、信息释放、情绪曲线。

### 操作指引

| 维度 | 分析内容 | 关键问题 |
|:-----|:---------|:---------|
| 叙事节奏 | 紧张↔舒缓的交替节奏 | 紧张场景和舒缓场景如何交替？ |
| 信息释放 | 世界观信息、角色信息、剧情信息的释放节奏 | 什么时候给新信息？给多少？ |
| 情绪曲线 | 读者情绪的高低起伏 | 情绪峰值在哪些节点？低谷在哪些节点？ |
| 对话/叙事/描写比例 | 三要素在全章中的分配 | 不同类型章节的配比有何规律？ |
| 切换技法 | 视角切换、场景切换、时间切换的频率和技法 | 切换是否流畅？节奏是否被打断？ |

### 分析方法

```
1. 选取3-5个代表性章节，逐段标注「叙事/对话/描写/心理」
   统计各类型占比
2. 绘制情绪曲线：从"1(平淡)"到"5(激动)"标注每1000字的情绪强度
3. 标注信息释放点：什么位置给出了什么信息
4. 分析切换频率：多少字有一次场景/视角/时间切换
```

### 输出

```
## 节奏拆解报告

### 选章分析
| 章节 | 叙事% | 对话% | 描写% | 心理% | 备注 |
|:-----|:------|:------|:------|:------|:-----|
| 第X章 | 35% | 40% | 15% | 10% | 紧张对峙章 |
| 第Y章 | 20% | 25% | 40% | 15% | 日常过渡章 |
| 第Z章 | 30% | 35% | 20% | 15% | 高潮爆发章 |

### 情绪曲线
```
情绪
5  ↑  ╱╲   ╱╲     ╱╲
4  ↑ ╱  ╲ ╱  ╲   ╱  ╲
3  ↑╱    ╲    ╲ ╱    ╲
2  ↑      ╲    ╲      ╲
1  └──────────────────────
   第1章        第X章        第Y章
```

### 信息释放节奏
| 位置 | 释放的信息 | 目的 |
|:-----|:----------|:-----|
| 第1章 | 世界基础设定 | 让读者快速理解世界 |
| 第3章 | 主角金手指 | 制造爽点期待 |
| 第10章 | 反派首次露面 | 建立中期威胁 |
| 每5章 | 新设定的微信息 | 保持新鲜感 |

### 节奏技法提炼
1. **[紧张-舒缓交替模式]**：______
   原文分析：______
   可复用技巧：______
```

| `webnovel-trend` | 扫榜环节可调用 `webnovel-trend` 技能辅助数据分析和市场定位 |

## 前置条件

- 需要有至少一部目标作品（全文或前30章以上文本）
- 拆解对象优先选择同赛道优秀作品
- 建议每次拆文聚焦1-2个学习目标，避免贪多
