# 错误知识库

> 灵境系统的"自省"能力——记录每一次错误，防止同类问题复发。
> 🔧 **管理归属：** `company/debug/`（修错部门）

## 结构

```
knowledge/errors/
├── README.md              ← 本文件（库门面）
├── root-causes.json       ← 机器可读的根因数据
├── entries/               ← 每次错误的完整记录
├── categories/            ← 7类根因的定义与对策模板
├── pre-checks/            ← 每类根因的预防脚本
└── learned/               ← 系统"学会"的自动化规则
```

## 7类根因速查

| # | 根因 | 发生次数 | 对策 |
|:-:|:-----|:--------:|:-----|
| 1 | 改了A不改B | 10 | 改后全局grep旧值 |
| 2 | 声明没实现 | 12 | 声明时确认文件存在 |
| 3 | 脚本静默失败 | 4 | 每条命令加错误处理 |
| 4 | 路径泄露 | 4 | 提交前搜绝对路径 |
| 5 | 创建不验证 | 3 | 建完必须执行一次 |
| 6 | 三层不同步 | 2 | 改审查必改模板 |
| 7 | 子智能体不闭环 | 7 | 交付前三问 |

## 如何添加新错误

1. 在 `entries/` 下创建 `YYYY-MM-DD_简短描述.md`
2. 填写模板（摘要/根因分析/对策/防复发/涉及文件）
3. 更新 `root-causes.json` 计数
4. 更新 `README.md` 速查表计数
