# 两阶段写作流程 — 创作(T=0.7)→结算(T=0.3)

> ⚠️ 本流程是灵境系统写作质量的核心保障。任何章节创作必须经历两阶段：**高熵创作**→**低熵结算**。
> 灵感阶段的"模糊面纱"必须在结算阶段被梳理为清晰连贯的叙事。先放得开，再收得拢。

## 一、核心思想

### 为什么需要两阶段？

单一温度（temperature）写作的问题：

| 单温度问题 | 表现 | 后果 |
|:-----------|:-----|:-----|
| **T始终偏高** | 文风飘逸、联想跳跃、旁逸斜出过多 | 主线模糊、节奏失控、信息密度低 |
| **T始终偏低** | 文风僵硬、缺乏灵气、叙事直白 | 干涩乏味、缺少惊艳感、可读性差 |
| **T折中** | 两头不靠——不够野也不够稳 | 平庸，缺乏个性 |

**两阶段方案**：先用高温度（T=0.7）产生丰富素材和意外之喜，再用低温度（T=0.3）收敛、结构化、去芜存菁。这是InkOS管线思想"先发散后收敛"在灵境中的落地。

### 温度分层：T值语义

| T值区间 | 语义 | 行为倾向 | 适用阶段 |
|:--------|:-----|:---------|:---------|
| 0.7-0.9 | 🎨 **高熵探索** | 词汇丰富、句法多变、联想自由、情节可选路径多 | 创作阶段 |
| 0.5-0.6 | 📐 **中等结构化** | 平衡创造性与连贯性 | 创作→结算过渡 |
| 0.2-0.4 | 🔍 **低熵收敛** | 稳定输出、选择最佳路径、去冗余、强化逻辑链 | 结算阶段 |
| 0.1-0.2 | ⚙️ **机械验证** | 极低温度，精确执行已知模式 | 审查/修改/校对 |

### 与温度互补的参数

| 参数 | 创作阶段（T=0.7） | 结算阶段（T=0.3） | 说明 |
|:-----|:------------------|:-----------------|:------|
| **top_p** | 0.9 | 0.7 | 高p保留更多候选词 |
| **top_k** | 50 | 20 | 宽窗选词vs窄窗精筛 |
| **frequency_penalty** | 0.3 | 0.1 | 创作期鼓励用新词 |
| **presence_penalty** | 0.3 | 0.0 | 结算期回归主题 |
| **max_tokens /思考深度** | 适当放宽 | 收紧到够用 | 创作期允许绕路探索 |

## 二、两阶段总览

```
┌─────────────────────────────────────────────────────┐
│                    两阶段写作流程                      │
│                                                       │
│  创作阶段 (T=0.7)             结算阶段 (T=0.3)         │
│  ┌─────────────────┐        ┌─────────────────┐       │
│  │ 发散探索          │  ──→  │ 收敛整合          │       │
│  │ • 生成3-5个开篇    │        │ • 选择最佳路径     │       │
│  │ • 探索多条情节路径 │        │ • 合并碎片为连贯体 │       │
│  │ • 保留纠结/矛盾   │        │ • 消灭逻辑矛盾     │       │
│  │ • 不自我审查      │        │ • 强制自检清单     │       │
│  └─────────────────┘        └─────────────────┘       │
│         │                          │                   │
│         └────── 过渡判据 ──────────┘                   │
│               • 素材量≥2000字                          │
│               • 关键节点≥2个触发                       │
│               • 无未决分支                              │
└─────────────────────────────────────────────────────┘
```

### 核心公式

```
写作质量 = 创作阶段的「可能性广度」× 结算阶段的「收敛精度」
```

缺少任一个阶段，另一个阶段的效果都无法最大化。

## 三、创作阶段（T=0.7）—— 高熵探索

### 3.1 目标

在低约束环境下产生尽可能丰富的叙事素材，包括但不限于：

- **情节可能性**：多个发展方向的可选路径
- **对话变体**：不同情绪基调下的对话版本
- **感官素材**：场景的视觉/听觉/嗅觉/触觉细节
- **意外关联**：跨伏笔的意外连接、角色间的意外互动
- **边缘想法**：可能不采用但值得记录的点子

### 3.2 产出物

| 产出物 | 格式 | 用途 | 字数要求 |
|:-------|:-----|:------|:---------|
| 素材池 | 带编号的片段清单（3-5个可选项 × 关键节点） | 结算阶段选择/组合 | 不设限 |
| 路径草稿 | 非线性片段（非完整章节，可能有重叠或矛盾） | 保留多种可能性 | ≥1500字 |
| 亮点标记 | `✨` 标记最满意的段落 | 结算阶段优先保留 | — |

### 3.3 写作指引

创作阶段的Agent提示词应包含：

```
【创作阶段（T=0.7）指引】
你现在处于高熵探索阶段。你的任务是：
1. 生成3-5个不同的开篇方向——至少一个有反转或意外
2. 对每个关键场景提供2种以上的写法（用"方案A/方案B"标注）
3. 大胆尝试非常规句式和词汇——不怕"过"
4. 标记你最满意的段落（✨）
5. 保留矛盾和不一致——结算阶段会处理
6. 如果某个边缘想法觉得"可能没用但有意思"——写下来
7. ⚠️ 不自我审查、不提前收敛、不优先考虑逻辑一致性
```

### 3.4 创作阶段判据：何时可以进入结算？

| 判据 | 说明 | 硬性条件 |
|:-----|:------|:--------|
| **素材充足** | 素材池总字数 ≥ 最终章节目标的150% | ✅ 硬性 |
| **关键节点触发** | 本章大纲中≥2个关键情节节点已有写法探索 | ✅ 硬性 |
| **无未决分支** | 大纲要求的每个要素都有至少1个写法覆盖 | ✅ 硬性 |
| **亮点密度** | 每500字至少有1个创作方自评的亮点段落 | ❌ 软性（可跳过但需注明原因） |

### 3.5 覆盖场景

创作阶段适用于以下所有写作Agent：

| Agent | 创作阶段的关注点 |
|:------|:-----------------|
| 写手Agent | 开篇方向多样性、场景描写创意、对话风格探索 |
| 角色设计师 | 角色行为的多种可能性、情感表达的不同方式 |
| 剧情架构师 | 伏笔埋设位置的多方案比较、剧情走向的多条路径 |
| 喜剧写手 | 笑点分布的多种节奏、缺陷引爆的不同时机 |
| 恋爱写手 | 感情递进的不同节奏模式、告白的变体写法 |
| 身份悬疑写手 | 信息披露节奏的多方案、误认层级的多种设计 |

## 四、结算阶段（T=0.3）—— 低熵收敛

### 4.1 目标

从创作阶段的丰富素材中选择最佳路径，收敛为一条清晰连贯的叙事线：

- **路径选择**：从多个开篇/走向中选择最优解
- **碎片合并**：将创作阶段的多个片段合并为流畅的线性叙事
- **逻辑打磨**：角色行为合理性、因果链完整性、时间线一致性
- **去冗余**：删除「发散痕迹」——方案标记、替代写法、边缘想法注释
- **去AI化**：应用 `anti-ai-polish` 四步流水线（检测→标记→改写→验证）

### 4.2 结算流程（5步）

```
第1步：素材评估与路径选择
  ├── 读创作阶段产出物
  ├── 对比各方案 → 选择最优路径（或组合路径）
  │   选择标准：
  │   ├── 与本章情感基调最匹配
  │   ├── 与前后章衔接最自然
  │   ├── 亮点保留率最高
  │   └── 角色行为合理性最强
  └── 输出：选择的路径 + 舍弃方案的摘要（留痕）

第2步：结构收敛
  ├── 将碎片合并为连贯章节结构
  ├── 补充过渡段落（创作阶段可能缺失的衔接）
  ├── 确保时间线连续
  └── 应用「叙事节奏控制」技法：松→紧→松的交替

第3步：精细化打磨（T=0.3环境）
  ├── 消灭所有逻辑矛盾
  ├── 统一叙述视角（不随意切换POV）
  ├── 删除√发散标记（方案A/B标注、✨标记、备注注释）
  ├── 优化句式：长句拆短、啰嗦删减、重复合并
  └── 评估字数：≥2000汉字

第4步：去AI化处理
  ├── 运行 anti-ai-polish 四步流水线
  │   ├── 检测：扫描AI味特征（完美对称句式/空洞过渡词/完美对话）
  │   ├── 标记：P0问题（强制改）/ P1-P2问题（记录到报告）
  │   ├── 改写：逐句修正
  │   └── 验证：二次扫描确认无残留
  └── 参考 knowledge/rules/common/dialogue-quality.md 10项检查

第5步：清算与输出
  ├── ⚠️ 执行结算自检清单（见4.4节）
  ├── 生成最终章节文件
  ├── ⚠️ 获取写锁 W-lock(chapter:{volume}:{num}) 后写入
  └── 释放锁 → 通知负责人「章节已落盘，可派审查官」
```

### 4.3 结算阶段的温度控制参数（推荐）

结算阶段并非全程T=0.3。细分子步骤的温度微调：

| 子步骤 | 推荐T值 | 说明 |
|:-------|:--------|:------|
| 素材评估与路径选择 | 0.3 | 判断力优先，温度低 |
| 结构收敛 | 0.3 | 逻辑性和衔接优先 |
| 精细化打磨（句式优化） | 0.25 | 更低温度，更精确 |
| 去AI化改写 | 0.2 | 机械性替换，不需要创意 |
| 过渡段/衔接句补充 | 0.35 | 稍高温度避免生硬 |
| 标题设计 | 0.4 | 需要创意但不过度发散 |

### 4.4 结算阶段自检清单（硬性）

每一章节结算完成后必须逐条确认：

| # | 检查项 | 通过条件 |
|:-:|:-------|:---------|
| 1 | ✅ 字数达标 | ≥2000汉字 |
| 2 | ✅ 无发散标记残留 | 无「方案A/B」「✨」「TODO」「备选」等标记 |
| 3 | ✅ 无逻辑矛盾 | 角色行为、时间线、因果链均自洽 |
| 4 | ✅ 无AI味P0问题 | anti-ai-polish 检测P0=0 |
| 5 | ✅ 破折号合规 | `——` 出现次数 ≤3（仅用于语音延长/对话打断） |
| 6 | ✅ 无章节号元引用 | 叙述和对话中无「第X章」字样 |
| 7 | ✅ 视角统一 | 同一场景内不切换POV |
| 8 | ✅ 与上一章衔接 | 开篇有连续性锚点（参照前一章末尾状态） |
| 9 | ✅ 与下一章预备 | 结尾自然引出下一章（非硬性截断） |
| 10 | ✅ 角色行为合理 | 每个角色的行为符合其性格/当前状态/动机 |

> 任何一项❌ → **禁止落盘** → 打回结算阶段的对应子步骤修正 → 重新检查。
> 连续2次结算检查仍未通过 → 报负责人人工介入。

## 五、两阶段与现有创作流程的集成

### 5.1 替换写手Agent的单阶段提示词

当前 `chapter-creation.md` 第2步的写手Agent是单阶段提示词：

```
写手智能体
├── 调用知识检索获取上下文
├── 参考角色设计师和剧情架构师的同步产出
├── 执行：正文写作 + 标题设计
└── 输出：完整章节正文 → 写入文件（已落盘）
```

两阶段替换为：

```
写手智能体（两阶段）
│
├── 【第一阶段：创作 T=0.7】
│   ├── 调用知识检索获取上下文（R锁：foreshadow, outline, character）
│   ├── 参考角色设计师和剧情架构师的同步产出
│   ├── 高熵探索：生成3-5个开篇方向 + 关键场景多方案
│   ├── 标记亮点（✨）
│   └── 输出：素材池（非线性片段，含多个方案）→ 临时文件
│
├── 过渡判定（素材充足？关键节点覆盖？无未决分支？）
│   ├── ✅ 满足 → 进入结算阶段
│   └── ❌ 不满足 → 继续创作阶段（最多2次补充探索）
│
├── 【第二阶段：结算 T=0.3】
│   ├── 素材评估与路径选择
│   ├── 结构收敛：合并碎片为线性叙事
│   ├── 精细化打磨：逻辑修正、句式优化
│   ├── 去AI化：运行 anti-ai-polish 流水线
│   ├── 执行结算自检清单（10项硬性检查）
│   └── 输出：完整章节正文（已通过自检）
│       └── ⚠️ 获取 W-lock(chapter:{volume}:{num}) → 写入文件 → 释放锁
```

### 5.2 与审查官流程的衔接

审查官不需要感知两阶段——它读的是结算阶段后的成品文件。但审查报告可增加一条维度：

**「两阶段痕迹」**：审查章节是否存在以下结算不彻底痕迹：
- ❌ 有发散标记残留（方案标注、TODO等）
- ❌ 有明显风格突变（高熵片段淹没在低熵叙事中）
- ❌ 前500字和后500字质量落差过大（创作阶段的粗放未收敛）

### 5.3 与角色设计师、剧情架构师的集成

角色设计师和剧情架构师同样可用两阶段模式（创作→结算）：

```
角色设计师（两阶段）
├── 创作阶段 T=0.7：探索角色情感/行为的2-3种可能性
├── 过渡判定
└── 结算阶段 T=0.3：选择最合理的角色状态 → 写入角色变更清单

剧情架构师（两阶段）
├── 创作阶段 T=0.7：探索伏笔埋设的多点可选位置
├── 过渡判定
└── 结算阶段 T=0.3：选择最佳伏笔方案 → 写入伏笔追踪
```

### 5.4 集成后创作流程全景

```
第1步：负责人把握大方向
  └── 确定本章情感基调（暖/冷/冷暖交替）

第2步：写作部门两阶段同步创作（不可拆分）
  │
  ├── 写手（两阶段 T=0.7 → 过渡 → T=0.3）
  │   ├── 🔐 获取锁（R: foreshadow, outline, character → W: chapter）
  │   ├── 创作阶段：素材池 + 多方案
  │   ├── 过渡判定
  │   └── 结算阶段：自检清单 → 落盘 → 🔓 释放锁
  │
  ├── 角色设计师（两阶段 T=0.7 → T=0.3）
  ├── 剧情架构师（两阶段 T=0.7 → T=0.3）
  └── 其他写手Agent（两阶段）

第3步：审查官审查
  ├── 只读成品（结算阶段已完成的反向指标）
  ├── 增加「两阶段痕迹」审查维度
  └── 通过/打回

第4步：负责人执行更新 + 汇总报告
  └── 报告中含两阶段数据：素材量/选择轮次/自检通过率
```

## 六、两阶段的数据指标

### 6.1 关键指标

| 指标 | 含义 | 目标值 | 采集方式 |
|:-----|:------|:------|:---------|
| **素材利用率** | 结算阶段保留的创作素材占比 | ≥40% | 素材池字数 vs 成品字数 |
| **亮点保留率** | 创作阶段✨标记保留到成品的比例 | ≥70% | 计数比对 |
| **收敛比** | 素材池字数 / 成品字数 | 1.2-2.0 | 自动统计 |
| **结算自检通过率** | 首次结算即通过的自检项比例 | ≥90% | 自检记录 |
| **温度切换次数** | 同一章节内创作/结算切换次数 | ≤2（含过渡） | 流程日志 |
| **发散痕迹残留** | 结算遗漏的发散标记数 | 0 | 审查官检测 |

### 6.2 异常模式

| 模式 | 指标表现 | 根因 | 对策 |
|:-----|:---------|:------|:------|
| **素材不足** | 素材利用率 > 80% 或 收敛比 < 1.1 | 创作阶段探索不充分 | 提高创作阶段T值至0.8，或延长探索时间 |
| **过度发散** | 素材利用率 < 20% 或 收敛比 > 3.0 | 创作阶段过于发散，结算阶段丢弃过多 | 创作阶段增加方向约束（不减少可能性但加强聚焦） |
| **结算纰漏** | 自检首次通过率 < 70% | 结算温度过高（T>0.4）或自检不认真 | 降低结算阶段T值至0.25，检查提示词是否含结算指令 |
| **思路中断** | 成品中有风格突变段落 | 创作→结算中间隔了其他任务 | 确保两阶段连续执行，中间不插入其他Agent工作 |

## 七、与InkOS管线设计的对照

| 维度 | InkOS | 灵境两阶段 |
|:-----|:------|:-----------|
| 阶段划分 | 3 stages: Outline → Draft → Polish | 2 stages: 创作(T=0.7) → 结算(T=0.3) |
| 温度策略 | Stage-level temperature | Phase-level + Sub-step micro-adjustment |
| 发散机制 | 明确要求 writer 在 Draft stage 做"branching" | 创作阶段生成3-5个方案 + 关键节点多写法 |
| 收敛机制 | Polish stage 做 "cohesion pass" | 结算阶段5步流程：评估→结构→打磨→去AI→清算 |
| 质量门 | 审查体系独立于pipeline | 10项结算自检清单（落盘前） + 审查官（落盘后） |
| 输出契约 | 结构化产出（frontmatter + 正文） | 素材池（创作阶段）→ 成文（结算阶段） |
| 可追溯性 | — | 素材利用率、亮点保留率等量化指标 |

## 八、实现路线图
