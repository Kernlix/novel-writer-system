# 创建不验证

## 定义
新增了文件/脚本但从未执行验证过

## 典型场景
- 新脚本建完不跑
- 新hook不测试触发
- 新文件不检查格式

## 自动检查
```bash
pyt non3 新增文件.py # 建完必须实际执行一次
```

## 历史同类型错误
- （暂无）

## 对策
1. 发生该类错误时，立即记录到 `knowledge/errors/entries/`
2. 更新 `root-causes.json` 计数
3. 更新 `README.md` 速查表
4. 将自动检查脚本写入 `pre-commit-check.sh`

## 系统教训
- 该类错误已发生 3 次，证明光靠记忆不可靠
- 必须通过自动化脚本+pre-commit检查来阻断
